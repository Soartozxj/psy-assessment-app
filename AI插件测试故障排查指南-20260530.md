## 🔍 可能的原因

### 原因1：未通过登录认证 ⚠️

**现象**：

- 打开 `admin-legacy.html` 后，页面显示**登录遮罩**

1. 打开 `http://localhost:8080/mini-app-h5/backend/admin-legacy.html`
2. 查看页面是否显示**登录界面**（深色背景 + 用户名/密码输入框）
3. 按 `F12` 打开开发者工具，查看 Console

**如果看不到这条日志** → 说明未通过认证，插件系统未初始化

1. 在登录界面输入用户名和密码（默认 `admin` / `123456`）
2. 点击"登录"按钮
3. 登录成功后，登录遮罩消失，显示管理界面
4. 此时再点击左侧菜单的"AI配置"

**验证方法**：

- 登录后，Console显示 `🚀 开始初始化插件系统...`
- 左侧菜单可以正常点击
- 点击"AI配置"后，页面切换到AI配置页面

### 原因3：`switchSection()` 函数未正确激活菜单项 ⚠️

**现象**：

- 点击左侧菜单后，页面内容切换了
- 但左侧菜单的**激活状态**没有更新（菜单项没有高亮）
- Console显示 `ℹ️ 切换到页面: aiConfig`，但菜单项没有高亮

**诊断方法**：

1. 按 `F12` 打开开发者工具
2. 切换到 Console 标签页
3. 点击左侧菜单的"AI配置"
4. 查看页面是否切换到AI配置页面
5. 查看左侧菜单的"AI配置"项是否高亮（有 `.active` 类）

**如果页面切换了，但菜单项没有高亮** → 说明 `switchSection()` 函数中的菜单激活逻辑有问题

**问题代码**（第216-222行）：

```javascript
// 激活对应的菜单项 - 精确匹配（避免「测评报告」等含 'scales' 的菜单项被误激活）
document.querySelectorAll('.menu-item').forEach((m) => {
  const onclick = m.getAttribute('onclick');
  if (onclick && onclick.trim() === `switchSection('${name}')`) {
    m.classList.add('active');
  }
});
```

**问题分析**：

- 这段代码检查的是 `onclick` 属性
- 但我们的菜单项使用的是 `data-action="switch-section"` 事件委托方式
- 菜单项**没有 `onclick` 属性**，所以这个逻辑不会执行

**解决方案**：

**方案A：修改 `switchSection()` 函数，增加 `data-section` 匹配逻辑**

打开 `admin-legacy.html`，找到第216-222行，修改为：

```javascript
// 激活对应的菜单项 - 支持 onclick 和 data-action 两种方式
document.querySelectorAll('.menu-item').forEach((m) => {
  // 方式1：检查 onclick 属性（兼容旧代码）
  const onclick = m.getAttribute('onclick');
  if (onclick && onclick.trim() === `switchSection('${name}')`) {
    m.classList.add('active');
    return;
  }

  // 方式2：检查 data-section 属性（事件委托方式）
  const section = m.getAttribute('data-section');
  if (section === name) {
    m.classList.add('active');
  }
});
```

**修改步骤**：

1. 打开 `/Users/rich/WorkBuddy/20260407113106/mini-app-h5/backend/admin-legacy.html`
2. 找到第216-222行
3. 替换为上面的新代码
4. 保存文件
5. 刷新浏览器（`Cmd + R` 或 `F5`）
6. 重新测试点击"AI配置"菜单

**验证方法**：

- 点击"AI配置"后，页面切换到AI配置页面
- 左侧菜单的"AI配置"项高亮显示（有 `.active` 类）
- Console显示 `ℹ️ 切换到页面: aiConfig`

### 原因4：插件未正确加载 ⚠️

**现象**：

- 页面切换到AI配置页面了
- 但AI配置页面显示**空白**或**无内容**
- Console显示 `ℹ️ 切换到页面: aiConfig`，但没有后续的 `🎯 AI插件开始执行...` 日志

**诊断方法**：

1. 按 `F12` 打开开发者工具
2. 切换到 Console 标签页
3. 点击左侧菜单的"AI配置"
4. 查看Console输出

**预期日志**：

```
ℹ️ 切换到页面: aiConfig
🎯 AI插件开始执行... {action: 'loadConfig'}
📂 加载AI配置...
✅ AI配置加载成功
✅ AI配置加载成功，已更新UI
```

**如果看不到这些日志** → 说明插件未正确加载，或者 `PluginLoader.get('ai')` 返回 `null`

**解决方案**：

1. 检查插件是否成功加载
   - 在 Console 中执行：
     ```javascript
     console.log(PluginLoader.getPluginsInfo());
     ```
     **预期结果**：返回包含 `ai` 插件的对象

2. 如果插件未加载，手动初始化插件系统
   - 在 Console 中执行：
     ```javascript
     await PluginLoader.init();
     ```
     **预期结果**：显示 `✅ 插件系统初始化完成`

3. 重新测试
   - 点击左侧菜单的"AI配置"
   - 查看Console是否显示插件执行日志

**如果问题仍然存在** → 打开故障排查文档，查看"原因5"

### 原因5：AI配置页面的HTML结构缺失 ⚠️

**现象**：

- 页面切换到AI配置页面了
- 但AI配置页面显示**空白**或**无内容**
- Console显示 `🎯 AI插件开始执行...`，但没有错误

**诊断方法**：

1. 按 `F12` 打开开发者工具
2. 切换到 Console 标签页
3. 点击左侧菜单的"AI配置"
4. 查看Console输出，确认没有错误
5. 切换到 **Elements** 标签页
6. 搜索 `id="section-aiConfig"`

**如果找不到这个元素** → 说明AI配置页面的HTML结构缺失

**解决方案**：

1. 检查 `admin-legacy.html` 中是否有 `id="section-aiConfig"` 这个元素

   ```bash
   grep -n 'id="section-aiConfig"' /Users/rich/WorkBuddy/20260407113106/mini-app-h5/backend/admin-legacy.html
   ```

   **预期结果**：

   ```
   5435:        <section class="page-section" id="section-aiConfig">
   ```

2. 如果元素存在，检查它是否有内容
   - 在 Console 中执行：
     ```javascript
     console.log(document.getElementById('section-aiConfig').innerHTML);
     ```
     **预期结果**：返回AI配置页面的HTML内容

3. 如果元素存在，但内容为空 → 说明AI配置页面的HTML结构缺失，需要补充

**如果问题仍然存在** → 需要查看完整的HTML文件，定位问题
