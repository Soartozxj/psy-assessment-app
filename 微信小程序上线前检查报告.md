## 4️⃣ 前端 index.html 上线前检查 ⚠️

### 4.1 console.log 调试代码（44 处）

前端有大量 `console.log` 调试输出，**上线前建议清理或降级**。

**清理策略建议**：
- 生产环境全局禁用 console：添加以下代码到页面顶部
  ```javascript
  if (location.protocol === 'https:' && !location.search.includes('debug=1')) {
    console.log = console.debug = console.info = function(){};
  }
  ```
- 或保留但用 `localStorage('psy_debug')` 开关控制

### 4.2 敏感信息

| 类型 | 位置 | 风险等级 |
|------|------|----------|
| `localhost:8080` URL 残留 | 第2行 HTML 注释 | 低（注释不影响运行） |
| Ollama 默认地址 `localhost:11434` | 第5249行 / 第5262行 | 低（云端模式下不会生效，因为云函数 ai-call 代理） |
| **HTML 注释中的 saved from url** | 第2行 `<!-- saved from url=(0053)http://localhost:8080/... -->` | 低（建议删除） |

### 4.3 外部依赖

前端页面是纯原生 HTML/CSS/JS，无外部 CDN 依赖，**不存在第三方服务中断风险**。

## 5️⃣ 后台 admin-legacy.html 上线前检查 ⚠️

### 5.1 console.log 调试代码（6 处）

数量较少，建议同样用上述策略处理。

### 5.2 localhost 引用

| 行号 | 内容 | 说明 |
|------|------|------|
| 1244 | `placeholder="http://localhost:11434"` | Ollama 配置 UI，合理 |
| 1377 | `value="http://localhost:3600/index.html"` | 本地预览地址展示，合理 |
| 5375 | `ollama: { baseUrl: 'http://localhost:11434' }` | 默认配置 fallback，合理 |
| 5377 | 同上 | 默认配置 fallback，合理 |
| 5393 | `cfg.ollama.baseUrl \|\| 'http://localhost:11434'` | 读取配置 fallback，合理 |
| 5434 | `baseUrl: ...value \|\| 'http://localhost:11434'` | 保存配置 fallback，合理 |
| 5524 | `cfg.ollama.baseUrl \|\| 'http://localhost:11434'` | AI 调用 fallback，合理 |

**结论**：这些 `localhost` 引用都是 Ollama 本地模型配置的默认值，**在云端模式下 AI 调用走 `ai-call` 云函数代理，这些代码不会执行到**，无需修改。

### 5.3 后台权限控制

后台页面目前**没有前端层面的权限验证**（任何人均可通过 URL 访问 `admin-legacy.html`）。  
- 当前方案中，敏感操作（数据同步等）由云函数 `ADMIN_OPENIDS` 验证
- 但云函数 `ADMIN_OPENIDS` 为空数组，**验证形同虚设**（见云函数问题）

## 7️⃣ 数据层适配 ✅

`shared-data.js` + `cloud-data.js` 双层设计合理：

- **本地模式**：纯 localStorage，零风险
- **云端模式**：CloudData 自动检测环境 → 调用云函数 → 失败自动回退本地
- **环境检测**：URL 参数 `env=cloud` / `__wxjs_environment` / UserAgent 三重检测
- **AI 配置**：云端模式暂时仍存 localStorage（有 TODO 标记），`ai-call` 云函数从 config 集合读取 API Key

**已知限制**：
- `saveAiConfig()` / `getAiConfig()` 暂未接入 config 集合（代码中标注了 TODO）
- 图片资源 IndexedDB 在 WebView 中可用但不在云端持久化
