## ⚡ 审查优先级

| 标记              | 含义                         | 必须修复？ | 响应时间 |
| ----------------- | ---------------------------- | ---------- | -------- |
| 🔴 **Blocker**    | 安全漏洞、数据丢失、逻辑错误 | ✅ 是      | 立即     |

## 🎯 核心模块快速审查

### 🧠 Scoring Engine

- [ ] grouped 父亲/母亲分组是否独立计分？（不能把双方分数合并）
- [ ] `dimKey` 分组匹配覆盖 `father_` / `mother_` / `f_` / `m_` / 中文前缀
- [ ] 反向计分 `transform` 是否覆盖所有逆向题？
- [ ] 矩阵题逐行求和是否漏行？
- [ ] 空答案、边界值是否安全处理？

### 🎯 Meta Prompt & AI 管道

- [ ] 变量替换后 SP 是否有结构断裂（`{answers}` 撑爆段落）？
- [ ] 是否出现 `{dimensions['xx']['score']}` 嵌套变量（正则会失效）？
- [ ] SP 是否多写了"输出格式""禁止表格"等冗余格式约束？
- [ ] 角色推导是否声明了量表性质 + 非诊断边界？

### 📊 Test Engine

- [ ] EMBU 风格 grouped 题芯片高亮是否支持 ID / 数字两种答案？
- [ ] grouped 分值列是否分开展示父亲/母亲（非合并为一个数）？
- [ ] `_generateGroupedAnswer` 随机值是否各分组独立？

### 🚢 部署

- [ ] 部署前 `node --check` + div 平衡 + JSON 有效性 三项全通过？
- [ ] 部署前备份已创建？
- [ ] 生产环境路径（`SCALES_JSON_PATH`）区分本地/生产？
- [ ] 部署后生产环境验证通过？

## 🔍 通用审查速查

### 安全性 🔒

- [ ] 用户输入是否直接拼接到 SQL / HTML？
- [ ] 密钥、Token 是否硬编码？
- [ ] 微信 openid 是否泄漏到前端 `console.log`？
- [ ] 是否缺少权限/登录检查？

### 正确性 ✅

- [ ] 边界条件：空值、0、负数、最大值？
- [ ] 错误处理：try-catch 是否覆盖关键路径？
- [ ] 类型陷阱：`"1" === 1` → false（用 `==` 或显式转换）
- [ ] 异步操作是否有 await / .catch？

### 可维护性 🔧

- [ ] 函数名/变量名是否清晰自解释？
- [ ] 超过 50 行的函数是否需要拆分？
- [ ] 重复代码是否该抽取？
