1. 系统架构

系统采用前后端分离架构，前端为 H5 单页面应用，后端为微信云开发云托管服务。

技术栈：原生 JavaScript + 腾讯云 CloudBase + DashScope AI
部署方式：build-deploy.py 构建 → 微信云开发静态托管上传
数据存储：localStorage（本地）+ 云端 API（线上）双写

架构图：
前端 H5 → shared-data.js → scoring-engine.js → cloud-data.js → cloud-api.js → 云托管服务

小程序壳 → web-view → H5 页面（env=cloud 模式）

📊 方案对比
评估维度 当前方案 方案 A：Tab 分组 方案 B：分区折叠
页面长度 过长（5 卡片全展开） 一屏展示，零滚动 一屏展示标题，按需展开
信息可见性 全部内容直接可见 需切换 Tab 才能看到 标题+摘要常驻可见
操作效率 无需切换，直接操作 频繁切换 Tab 略繁琐 常用项展开，不常用折叠
改动量 — 中等（加 Tab 容器 + 分组） 中等（加折叠逻辑 + 双列）
推荐场景 信息量少时 模块独立、不常互操作时 频繁查看关键指标、按需查阅时
