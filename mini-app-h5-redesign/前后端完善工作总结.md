## 📊 前后端数据一致性工作流程

### 1. **后端操作流程**

```
管理员在后端页面操作 → 修改量表数据 → 共享数据模块保存 → 自动同步到前端 → 前端用户看到更新
```

### 2. **数据同步流程**

```
后端修改数据 → 触发syncToFrontend() → 数据写入前端key → 前端页面读取新数据 → 用户界面更新
```

### 3. **一致性监控流程**

```
系统运行 → 数据监控模块检查 → 发现不一致 → 显示警告 → 提供修复建议 → 管理员修复 → 重新检查
```

### 4. **故障排查流程**

```
发现数据问题 → 打开同步测试页面 → 运行所有测试 → 查看测试结果 → 执行修复操作 → 确认修复成功
```

## 🛠️ 使用指南

### 1. **日常操作指南**

#### 后端管理员：

- 打开 `backend/admin-legacy.html`
- 在量表管理页面修改数据
- 系统会自动同步到前端
- 点击"📊 数据监控"查看一致性状态

#### 前端用户：

- 打开 `frontend/index.html`
- 浏览最新的量表数据
- 点击"📊 监控"查看数据状态
- 点击"⚙️ 管理"进入后台管理

### 2. **测试验证指南**

#### 完整测试流程：

1. 打开 `data-sync-test.html`
2. 点击"测试连接"验证数据模块
3. 点击"检查一致性"验证前后端数据
4. 点击"测试同步"验证同步功能
5. 点击"测试操作"验证CRUD功能
6. 点击"加载数据"查看所有量表
7. 点击"显示监控面板"查看系统状态

### 3. **故障排查指南**

#### 数据不一致问题：

1. 打开监控面板查看不一致警告
2. 根据建议点击"同步到前端"
3. 重新加载页面检查数据
4. 如有持续问题，使用测试页面全面检查

#### 操作失败问题：

1. 使用测试页面验证各个功能
2. 检查浏览器localStorage数据
3. 使用"清除所有数据"重置系统
4. 重新加载默认数据测试
