## 第二部分：内容红线指南

### 🚨 核心原则

```
任何时候都不能触碰的三条底线：
① 国家法律法规
② 平台社区规范
③ 心理健康行业的专业伦理
```

---

### 🔴 红色禁区（绝对不能碰）

#### 1. 医疗诊断类（违法+违规）

| ❌ 绝对不能用             | ✅ 替代说法                        |
| ------------------------- | ---------------------------------- |
| "诊断出你有抑郁症/焦虑症" | "你的测评结果显示情绪状态需要注意" |
| "治疗你的心理问题"        | "帮你更好地了解自己的心理状态"     |
| "治愈/康复/痊愈"          | "改善/调整/提升"                   |
| "患者/病人"               | "用户/来访者/体验者"               |
| "心理咨询/心理治疗"       | "心理测评/心理探索/心理陪伴"       |

**法律依据**：《精神卫生法》——非医疗机构不得从事心理治疗和精神障碍诊断。

#### 2. 绝对化承诺（违规）

| ❌ 绝对不能用      | ✅ 替代说法              |
| ------------------ | ------------------------ |
| "保证让你不再焦虑" | "帮助你了解焦虑的来源"   |
| "100%准确"         | "提供科学的参考视角"     |
| "测一次就解决"     | "持续关注自己的心理状态" |

#### 3. 隐私侵犯（违法）

- ❌ 不得展示其他用户的测评数据（即使脱敏也需要用户授权）
- ❌ 不得要求用户提供完整的个人身份信息作为互动条件
- ❌ 不得在内容中暗示"不测就有问题"（制造焦虑）

**法律依据**：《个人信息保护法》——处理个人信息需取得个人同意。

#### 4. 歧视与伤害（违法+违规）

- ❌ 不得贬低特定人群的人格特质（"XX人格就是有问题"）
- ❌ 不得制造群体对立（"内向的人注定..."）
- ❌ 不得鼓励自伤/自杀行为（零容忍）
- ❌ 不得使用恐惧营销（"你再不管就晚了"）

### 🟡 黄色警戒（需要谨慎处理）

#### 1. 用户案例分享

```
✅ 可以写：用户故事（经脱敏+授权）
✅ 可以写：虚构的典型案例（标注"为保护隐私，案例已做虚构处理"）
❌ 不可以写：真实的用户测评数据（即使脱敏）
❌ 不可以写："我有一个用户xxx"（暗示真实案例）
```

#### 2. 量表科学性描述

```
✅ 可以写："基于国际通用的心理学量表"
✅ 可以写："经过大量样本验证"
❌ 不要写："临床验证""医院使用"
❌ 不要写具体量表名称（如SDS、SAS）暗示诊断用途
```

#### 3. 与竞品对比

```
✅ 可以写："星蓝心镜特色是AI对话式测评体验"
❌ 不要写："比XX产品更准/更好"
❌ 不要点名贬低任何竞品
```

#### 4. AI角色定位

```
✅ 可以写："AI心理咨询师/AI心理陪伴者"
✅ 可以写："AI辅助分析"
❌ 不要写："AI医生/AI治疗师"

```
