## 🎯 核心原则

### 1.1 一致性原则（最高优先级）

**规则**: 在同一个项目中，代码风格必须保持一致。

**理由**:

- 提高代码可读性
- 减少团队协作成本
- 降低代码审查难度

**示例**:

```javascript
// ✅ 正确：整个项目统一使用单引号
const name = 'John';
const message = 'Hello, ' + name;

// ❌ 错误：混用单引号和双引号
const name = 'John';
const message = 'Hello, ' + name;
```

### 1.2 可读性优先原则

**规则**: 代码是写给人看的，其次才是给机器执行的。

**理由**:

- 代码阅读次数远多于编写次数
- 清晰的代码减少Bug产生
- 便于代码审查和协作

### 1.3 ES6+ 优先原则

**规则**: 优先使用ES6+语法，禁止使用ES5过时语法。

**理由**:

- ES6+ 语法更简洁、更安全
- 项目已确定使用ES6+ (见《代码规范与规则.md》)
- 浏览器支持已不再是问题（Chrome 60+, Firefox 55+, Safari 11+）

## 🎨 代码格式化规则

### 2.1 缩进规则（强制）

**规则**: 使用 **2个空格** 进行缩进，禁止使用Tab字符。

**理由**:

- 2空格在JavaScript社区中更常见
- 避免不同编辑器Tab宽度不一致问题
- 使代码更紧凑，减少换行

**示例**:

```javascript
// ✅ 正确：2个空格缩进
class PluginBase {
  constructor(config = {}) {
    this.name = config.name || '未命名插件';
    if (this.name) {
      console.log('插件创建成功');
    }
  }
}

// ❌ 错误：使用Tab或4个空格
class PluginBase {
  constructor(config = {}) {
    // 4个空格，错误
    this.name = config.name; // Tab字符，错误
  }
}
```

**编辑器配置**:

```json
// .editorconfig
[*.js]
indent_style = space
indent_size = 2
```

### 2.2 引号规则（强制）

**规则**: 统一使用 **单引号** (`'`) 定义字符串。

**理由**:

- JavaScript社区主流风格
- JSON中使用双引号，区分代码和数据
- 减少Shift键使用，提高输入效率

**示例**:

```javascript
// ✅ 正确：使用单引号
const name = 'John';
const message = 'Hello, ' + name;
const template = `<div class="container">${name}</div>`;

// ❌ 错误：使用双引号
const name = 'John'; // 错误
const message = 'Hello, ' + name; // 错误
```

**例外情况**:

- JSON数据中必须使用双引号（JSON标准要求）
- 模板字符串中使用反引号（`` ` ``）

### 2.3 分号规则（强制）

**规则**: **必须** 在每条语句末尾添加分号。

**理由**:

- 避免JavaScript自动分号插入(ASI)的陷阱
- 提高代码清晰度
- 减少潜在Bug

**示例**:

```javascript
// ✅ 正确：每条语句都以分号结尾
const name = 'John';
console.log(name);
function greet() {
  return 'Hello';
}

// ❌ 错误：省略分号
const name = 'John'; // 缺少分号
console.log(name); // 缺少分号
```

### 2.4 空格规则（强制）

#### 2.4.1 操作符两侧空格

**规则**: 在操作符（`=`, `==`, `===`, `+`, `-`, `&&`, `||` 等）两侧各添加一个空格。

**示例**:

```javascript
// ✅ 正确：操作符两侧有空格
const sum = a + b;
if (age >= 18 && age <= 65) {
  console.log('工作日');
}

// ❌ 错误：操作符两侧缺少空格
const sum = a + b; // 错误
if (age >= 18 && age <= 65) {
  // 错误
}
```

#### 2.4.2 逗号后空格

**规则**: 在逗号（`,`）后添加一个空格，逗号前不加空格。

**示例**:

```javascript
// ✅ 正确：逗号后有空格
const person = { name: 'John', age: 30 };
const numbers = [1, 2, 3, 4];
function greet(name, age) {
  // ...
}

// ❌ 错误：逗号后缺少空格
const person = { name: 'John', age: 30 }; // 错误
const numbers = [1, 2, 3, 4]; // 错误
```

#### 2.4.3 关键字后空格

**规则**: 在关键字（`if`, `else`, `for`, `while`, `function`, `return`, `class`, `import`, `export` 等）后添加一个空格。

**示例**:

```javascript
// ✅ 正确：关键字后有空格
if (condition) {
  // ...
} else {
  // ...
}

for (let i = 0; i < 10; i++) {
  // ...
}

function greet(name) {
  return 'Hello, ' + name;
}

// ❌ 错误：关键字后缺少空格
if (condition) {
  // 错误
}
for (let i = 0; i < 10; i++) {
  // 错误
}
```

### 2.5 换行规则（推荐）

#### 2.5.1 最大行长度

**规则**: 每行代码不超过 **100个字符**。超过时应换行。

**理由**:

- 避免横向滚动
- 便于并排对比代码
- 提高代码可读性

**示例**:

```javascript
// ✅ 正确：换行保持可读性
const longMessage = '这是一个很长的字符串，超过100个字符时需要换行，' + '确保代码保持良好可读性。';

// ✅ 正确：使用模板字符串
const longMessage = `这是一个很长的字符串，超过100个字符时需要换行，
                    确保代码保持良好可读性。`;

// ❌ 错误：一行过长
const longMessage = '这是一个很长的字符串，超过100个字符时需要换行，确保代码保持良好可读性。';
```

#### 2.5.2 函数/方法换行

**规则**: 当函数参数超过3个或行长超过100字符时，每个参数单独占一行。

**示例**:

```javascript
// ✅ 正确：参数过多时换行
function createPlugin(name, version, description, config) {
  // ...
}

// ✅ 正确：函数调用时参数换行
const result = someLongFunctionName(argument1, argument2, argument3, argument4);

// ❌ 错误：一行过长
function createPlugin(name, version, description, config) {
  // 过长
}
```

#### 2.5.3 链式调用换行

**规则**: 链式调用（`.then()`, `.catch()`, `.forEach()` 等）每个方法调用单独占一行。

**示例**:

```javascript
// ✅ 正确：链式调用换行
fetchData()
  .then((response) => response.json())
  .then((data) => processData(data))
  .catch((error) => console.error(error));

// ✅ 正确：数组方法链式调用
const result = array
  .filter((item) => item.active)
  .map((item) => item.name)
  .join(', ');

// ❌ 错误：链式调用写在一行
fetchData()
  .then((response) => response.json())
  .then((data) => processData(data))
  .catch((error) => console.error(error));
```

### 2.6 大括号规则（强制）

**规则**: 使用 **K&R风格**（开括号在同一行，闭括号独占一行）。

**理由**:

- JavaScript社区主流风格
- 减少代码行数
- 提高代码密度

**示例**:

```javascript
// ✅ 正确：K&R风格
if (condition) {
  // 代码块
} else {
  // 代码块
}

function greet(name) {
  return 'Hello, ' + name;
}

class Plugin {
  constructor() {
    // ...
  }
}

// ❌ 错误：Allman风格（开括号独占一行）
if (condition) {
  // 代码块
}

function greet(name) {
  return 'Hello, ' + name;
}
```

### 2.7 空行规则（推荐）

**规则**: 使用空行分隔逻辑相关的代码块，提高可读性。

**建议**:

1. 在方法/函数之间添加1个空行
2. 在逻辑段落之间添加1个空行
3. 在 import 语句和代码之间添加1个空行
4. 禁止连续添加2个以上空行

**示例**:

```javascript
// ✅ 正确：合理使用空行
import { PluginBase } from './plugin-base.js';

class AIPlugin extends PluginBase {
  constructor() {
    super();
    this.name = 'AI Plugin';
  }

  async init() {
    // 初始化配置
    await this.loadConfig();

    // 测试连接
    await this.testConnection();
  }

  async execute(params) {
    // ...
  }
}

// ❌ 错误：空行使用不当
import { PluginBase } from './plugin-base.js';

class AIPlugin extends PluginBase {
  constructor() {
    super();

    this.name = 'AI Plugin';
  }
}
```
