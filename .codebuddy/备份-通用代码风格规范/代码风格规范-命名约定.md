## 🏷️ 命名约定

### 3.1 变量命名（强制）

**规则**: 使用 **camelCase（小驼峰）** 命名变量。

**理由**:

- JavaScript社区标准
- 与内置API保持一致（如 `innerHTML`, `className`）

**示例**:

```javascript
// ✅ 正确：camelCase命名
const firstName = 'John';
const lastName = 'Doe';
const isInitialized = false;
const maxRetryCount = 3;

// ❌ 错误：其他命名风格
const first_name = 'John'; // 下划线命名，错误
const FIRSTNAME = 'John'; // 常量命名用于变量，错误
const firstname = 'John'; // 全小写，可读性差
```

**特殊规则**:

- 布尔值变量使用 `is`, `has`, `can`, `should` 等前缀
- 私有变量使用 `_` 前缀（但不强制）

```javascript
// ✅ 正确：布尔值命名
const isActive = true;
const hasPermission = false;
const canEdit = true;
const shouldUpdate = false;

// ✅ 正确：私有变量约定
class Plugin {
  constructor() {
    this._internalState = {}; // 私有变量
    this.publicState = {}; // 公共变量
  }
}
```

### 3.2 常量命名（强制）

**规则**: 使用 **UPPER_SNAKE_CASE（大写蛇形）** 命名常量。

**理由**:

- 与JavaScript内置常量保持一致（如 `Math.PI`, `Infinity`）
- 清晰标识这是不可变的值

**示例**:

```javascript
// ✅ 正确：常量使用UPPER_SNAKE_CASE
const MAX_RETRY_COUNT = 3;
const API_BASE_URL = 'https://api.example.com';
const DEFAULT_CONFIG = {
  timeout: 5000,
  retry: 3
};

// ❌ 错误：常量使用camelCase
const maxRetryCount = 3; // 错误
const apiBaseUrl = 'https://api.example.com'; // 错误
```

**注意**:

- 使用 `const` 声明的变量不一定是常量（如 `const obj = { ... }` 可以修改属性）
- 只有真正不可变的值才使用UPPER_SNAKE_CASE

### 3.3 函数命名（强制）

**规则**: 使用 **camelCase（小驼峰）** 命名函数，动词前缀。

**理由**:

- 与变量命名保持一致
- 动词前缀清晰表达函数意图

**示例**:

```javascript
// ✅ 正确：函数命名
function getUserInfo() { ... }
function calculateTotal() { ... }
function isValidEmail() { ... }
function renderUI() { ... }

// ❌ 错误：函数命名
function getuserInfo() { ... }  // 首字母应小写
function CalculateTotal() { ... }  // 首字母应小写
function validEmail() { ... }     // 缺少动词前缀
```

**推荐动词前缀**:
| 动词前缀 | 含义 | 示例 |
|---------|------|------|
| `get` | 获取（有返回值） | `getUser()`, `getConfig()` |
| `set` | 设置 | `setUser()`, `setConfig()` |
| `is`/`has`/`can` | 判断（返回布尔值） | `isValid()`, `hasPermission()` |
| `load`/`fetch` | 加载数据 | `loadData()`, `fetchUser()` |
| `save`/`store` | 保存数据 | `saveData()`, `storeConfig()` |
| `render`/`display` | 渲染/显示 | `renderUI()`, `displayModal()` |
| `handle`/`on` | 事件处理 | `handleClick()`, `onSubmit()` |
| `init`/`create` | 初始化/创建 | `initPlugin()`, `createElement()` |
| `destroy`/`remove` | 销毁/删除 | `destroyPlugin()`, `removeElement()` |

### 3.4 类命名（强制）

**规则**: 使用 **PascalCase（大驼峰）** 命名类。

**理由**:

- 与JavaScript内置类保持一致（如 `Date`, `Array`, `Map`）
- 清晰标识这是类（构造函数）

**特殊规则（插件类）**:

- 插件类**必须**添加 `Plugin` 后缀（详见《Skills架构改造-代码结构规范》第2.2节）
- 示例：`AIPlugin`, `ScalePlugin`, `AuthPlugin`

**示例**:

```javascript
// ✅ 正确：类命名
class PluginBase { ... }
class AIPlugin extends PluginBase { ... }  // 插件类加Plugin后缀
class EventHub { ... }

// ❌ 错误：类命名
class pluginBase { ... }  // 首字母应大写
class AiPlugin { ... }    // 应使用AIPlugin（插件类）
class AIP { ... }         // 插件类必须加Plugin后缀
```

### 3.5 方法命名（强制）

**规则**: 使用 **camelCase（小驼峰）** 命名类的方法。

**示例**:

```javascript
// ✅ 正确：方法命名
class PluginBase {
  constructor() { ... }
  async init() { ... }
  async execute(params) { ... }
  destroy() { ... }
}

// ❌ 错误：方法命名
class PluginBase {
  Constructor() { ... }  // 应使用constructor（小写）
  Init() { ... }          // 首字母应小写
}
```

### 3.6 文件名命名（强制）

**规则**: 根据文件内容类型使用不同的命名规范。

| 文件类型         | 命名规范          | 示例                                 |
| ---------------- | ----------------- | ------------------------------------ |
| **类文件**       | kebab-case + .js  | `plugin-base.js`, `ai-plugin.js`     |
| **工具函数文件** | kebab-case + .js  | `ui-utils.js`, `string-helpers.js`   |
| **配置文件**     | kebab-case + .js  | `api-config.js`, `app-settings.js`   |
| **组件文件**     | kebab-case + .js  | `admin-modal.js`, `scale-card.js`    |
| **样式文件**     | kebab-case + .css | `admin-style.css`, `plugin-base.css` |

**特殊规则（插件文件）**:

- 插件文件**必须**遵循 `{插件名}-plugin.js` 格式（详见《Skills架构改造-代码结构规范》第2.2节）
- 示例：`ai-plugin.js`, `scale-plugin.js`, `auth-plugin.js`

**理由**:

- kebab-case在文件系统中最安全（不区分大小写）
- 与URL路径命名保持一致
- 提高文件可读性
- 插件文件命名规范化，便于自动加载

**示例**:

```javascript
// ✅ 正确：文件名命名
plugin - base.js;
ai - plugin.js; // 插件文件
ui - utils.js;
api - config.js;

// ❌ 错误：文件名命名
PluginBase.js; // 错误
ai_plugin.js; // 错误
uiUtils.js; // 错误
AIPlugin.js; // 错误：插件文件必须加-plugin后缀
```

### 3.7 事件命名（强制）

**规则**: 使用 **kebab-case** 命名自定义事件。

**理由**:

- 与HTML属性命名保持一致（如 `data-action="save-scale"`）
- 与DOM事件命名保持一致（如 `click`, `submit`, `mouseover`）

**特殊规则（插件事件）**:

- 插件事件命名**必须**遵循 kebab-case 格式（详见《Skills架构改造-代码结构规范》第2.2节）
- 示例：`plugin-loaded`, `user-login`, `data-saved`

**示例**:

```javascript
// ✅ 正确：事件命名
EventHub.on('plugin-loaded', callback);
EventHub.on('user-login', callback);
EventHub.on('data-saved', callback);

// ❌ 错误：事件命名
EventHub.on('pluginLoaded', callback); // 错误
EventHub.on('user_login', callback); // 错误
```

**参考**: 插件事件命名规范详见《Skills架构改造-代码规范与规则.md》第2.2节
