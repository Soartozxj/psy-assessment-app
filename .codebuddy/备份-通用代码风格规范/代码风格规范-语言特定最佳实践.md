## 💡 语言特定最佳实践

### 5.1 变量声明

**规则**:

1. 优先使用 `const` 声明变量
2. 只有需要重新赋值的变量才使用 `let`
3. **禁止使用 `var`**

**理由**:

- `const` 防止意外重新赋值
- `let` 有块级作用域，更安全
- `var` 有变量提升和函数作用域问题

**示例**:

```javascript
// ✅ 正确：使用const和let
const API_URL = 'https://api.example.com'; // 常量用const
const user = getUser(); // 不会被重新赋值用const

let count = 0; // 需要重新赋值用let
for (let i = 0; i < 10; i++) {
  count += i;
}

// ❌ 错误：使用var
var API_URL = 'https://api.example.com'; // 错误：应使用const
var count = 0; // 错误：应使用let
```

### 5.2 对象与数组

#### 对象创建

**规则**: 使用对象字面量 `{}` 创建对象，而不是 `new Object()`。

**示例**:

```javascript
// ✅ 正确：使用对象字面量
const user = {
  name: 'John',
  age: 30
};

// ❌ 错误：使用new Object()
const user = new Object(); // 错误
user.name = 'John';
user.age = 30;
```

#### 数组创建

**规则**: 使用数组字面量 `[]` 创建数组，而不是 `new Array()`。

**示例**:

```javascript
// ✅ 正确：使用数组字面量
const numbers = [1, 2, 3, 4, 5];

// ❌ 错误：使用new Array()
const numbers = new Array(1, 2, 3, 4, 5); // 错误
```

#### 解构赋值

**规则**: 使用解构赋值提取对象和数组的值。

**示例**:

```javascript
// ✅ 正确：使用解构赋值
const user = { name: 'John', age: 30, email: 'john@example.com' };
const { name, age, email } = user;

const numbers = [1, 2, 3, 4, 5];
const [first, second, ...rest] = numbers;

// ✅ 正确：函数参数解构
function greet({ name, age }) {
  return `Hello, ${name} (${age}岁)`;
}

// ❌ 错误：不使用解构赋值
const user = { name: 'John', age: 30 };
const name = user.name; // 繁琐
const age = user.age;
```

#### 扩展运算符

**规则**: 使用扩展运算符 `...` 复制对象和数组。

**示例**:

```javascript
// ✅ 正确：使用扩展运算符
const user = { name: 'John', age: 30 };
const updatedUser = { ...user, age: 31 }; // 复制并修改

const numbers = [1, 2, 3];
const newNumbers = [...numbers, 4, 5]; // 复制并添加

// ❌ 错误：直接修改原对象
const user = { name: 'John', age: 30 };
const updatedUser = user; // 错误：这是引用，不是复制
updatedUser.age = 31; // 会修改原对象
```

### 5.3 字符串

#### 模板字符串

**规则**: 使用模板字符串拼接字符串，而不是 `+` 运算符。

**示例**:

```javascript
// ✅ 正确：使用模板字符串
const name = 'John';
const age = 30;
const message = `Hello, ${name}! You are ${age} years old.`;

// 多行字符串
const html = `
  <div class="user">
    <h2>${name}</h2>
    <p>Age: ${age}</p>
  </div>
`;

// ❌ 错误：使用+运算符
const name = 'John';
const age = 30;
const message = 'Hello, ' + name + '! You are ' + age + ' years old.'; // 繁琐

// 多行字符串
const html =
  '<div class="user">\n' + // 繁琐
  '  <h2>' +
  name +
  '</h2>\n' +
  '  <p>Age: ' +
  age +
  '</p>\n' +
  '</div>';
```

### 5.4 函数

#### 箭头函数

**规则**: 优先使用箭头函数，特别是回调函数。

**理由**:

- 箭头函数更简洁
- 自动绑定 `this`，避免 `self = this` 的问题

**示例**:

```javascript
// ✅ 正确：使用箭头函数
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map((n) => n * 2);

// 多行箭头函数
const processed = numbers.map((n) => {
  const result = n * 2;
  return result;
});

// ❌ 错误：使用普通函数
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(function (n) {
  // 繁琐
  return n * 2;
});
```

**例外情况**:

- 需要使用 `this` 的函数（如对象方法）使用普通函数
- 需要提升的函数使用普通函数

#### 默认参数

**规则**: 使用默认参数，而不是在函数体内检查。

**示例**:

```javascript
// ✅ 正确：使用默认参数
function greet(name = 'Guest') {
  return `Hello, ${name}!`;
}

// ❌ 错误：在函数体内检查
function greet(name) {
  name = name || 'Guest'; // 旧式写法
  return `Hello, ${name}!`;
}
```

#### 剩余参数

**规则**: 使用剩余参数 `...` 代替 `arguments` 对象。

**示例**:

```javascript
// ✅ 正确：使用剩余参数
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}

// ❌ 错误：使用arguments对象
function sum() {
  const numbers = Array.from(arguments); // 繁琐
  return numbers.reduce((total, n) => total + n, 0);
}
```

### 5.5 类

#### 类定义

**规则**: 使用 `class` 语法定义类，而不是原型继承。

**示例**:

```javascript
// ✅ 正确：使用class语法
class PluginBase {
  constructor(config = {}) {
    this.name = config.name || '未命名插件';
  }

  async init() {
    // ...
  }
}

// ❌ 错误：使用原型继承
function PluginBase(config = {}) {
  this.name = config.name || '未命名插件';
}

PluginBase.prototype.init = function () {
  // ...
};
```

#### 继承

**规则**: 使用 `extends` 实现继承，使用 `super()` 调用父类构造函数。

**示例**:

```javascript
// ✅ 正确：使用extends
class AIPlugin extends PluginBase {
  constructor() {
    super({ name: 'AI Plugin' });
    this.model = 'qwen-turbo';
  }

  async init() {
    await super.init();
    // AI插件特定的初始化
  }
}

// ❌ 错误：手动设置prototype
function AIPlugin() {
  PluginBase.call(this, { name: 'AI Plugin' });
  this.model = 'qwen-turbo';
}

AIPlugin.prototype = Object.create(PluginBase.prototype);
AIPlugin.prototype.constructor = AIPlugin;
```

**插件类定义特殊说明**:

- 插件类**必须**遵循《Skills架构改造-代码结构规范》第2.3节的代码结构模板
- 插件类**必须**实现 `onInit()`, `onExecute()`, `onDestroy()` 等生命周期方法
- 插件私有属性**必须**加下划线前缀（如 `this._data`）
- 详细插件代码模板参见：《Skills架构改造-代码结构规范》第2.3节

### 5.6 异步编程

#### Promise

**规则**: 使用 `async/await` 代替 `.then()/.catch()`。

**理由**:

- `async/await` 更直观，像同步代码
- 错误处理更简单（使用 `try-catch`）
- 避免回调地狱

**示例**:

```javascript
// ✅ 正确：使用async/await
async function fetchUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('获取用户数据失败:', error);
    throw error;
  }
}

// ❌ 错误：使用.then()/.catch()
function fetchUserData(userId) {
  return fetch(`/api/users/${userId}`)
    .then((response) => response.json())
    .then((data) => data)
    .catch((error) => {
      console.error('获取用户数据失败:', error);
      throw error;
    });
}
```

#### 错误处理

**规则**: 所有异步函数必须使用 `try-catch` 进行错误处理。

**示例**:

```javascript
// ✅ 正确：使用try-catch
async function fetchData() {
  try {
    const data = await fetch('/api/data');
    return data;
  } catch (error) {
    console.error('获取数据失败:', error);
    // 可以在这里进行错误恢复或上报
    throw error; // 重新抛出，让调用者知道
  }
}

// ❌ 错误：缺少错误处理
async function fetchData() {
  const data = await fetch('/api/data'); // 可能抛出异常
  return data;
}
```

### 5.7 模块

#### 导入/导出

**规则**: 使用 ES6 模块语法（`import`/`export`），禁止使用 CommonJS（`require`/`module.exports`）。

**理由**:

- 项目已确定使用原生ES6+（无构建工具）
- 浏览器原生支持 `import()`
- 静态分析更友好

**示例**:

```javascript
// ✅ 正确：使用ES6模块语法
// 导出
export default class PluginBase { ... }
export function helper() { ... }

// 导入
import PluginBase from './plugin-base.js';
import { helper } from './helpers.js';

// ❌ 错误：使用CommonJS
module.exports = class PluginBase { ... }  // 错误
const PluginBase = require('./plugin-base.js');  // 错误
```

#### 动态导入

**规则**: 使用动态 `import()` 按需加载模块。

**示例**:

```javascript
// ✅ 正确：使用动态import()
async function loadPlugin(pluginName) {
  try {
    const pluginModule = await import(`./plugins/${pluginName}-plugin.js`);
    return pluginModule.default;
  } catch (error) {
    console.error('加载插件失败:', error);
    throw error;
  }
}

// ❌ 错误：使用require()
function loadPlugin(pluginName) {
  const pluginModule = require(`./plugins/${pluginName}-plugin.js`); // 错误
  return pluginModule;
}
```
