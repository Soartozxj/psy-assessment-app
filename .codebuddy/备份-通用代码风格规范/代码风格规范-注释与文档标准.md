## 📝 注释与文档标准

### 4.1 注释原则

**核心原则**: 注释应该解释 **为什么**，而不是 **是什么**。

**规则**:

1. 代码能自解释的，不加注释
2. 复杂逻辑必须加注释
3. 公共API必须加JSDoc注释
4. 临时修复或Workaround必须加注释说明原因

### 4.2 单行注释

**规则**: 使用 `//` 进行单行注释，在注释前添加一个空格。

**示例**:

```javascript
// ✅ 正确：单行注释
// 计算用户年龄
const age = currentYear - birthYear;

// 检查用户是否有权限
if (user.hasPermission) {
  // 执行敏感操作
  performSensitiveAction();
}

// ❌ 错误：单行注释
//计算用户年龄  // 缺少空格
const age = currentYear - birthYear;

//执行敏感操作  // 注释应放在代码上方，而不是同一行
if (user.hasPermission) {
  performSensitiveOperation();
}
```

### 4.3 多行注释

**规则**: 使用 `/* */` 进行多行注释，或在每行使用 `//`。

**示例**:

```javascript
// ✅ 正确：多行注释（方式1 - 推荐）
// 这是一个多行注释
// 用来解释复杂的业务逻辑
// 包括多个步骤和注意事项
function complexLogic() {
  // ...
}

// ✅ 正确：多行注释（方式2）
/*
 * 这是一个多行注释
 * 用来解释复杂的业务逻辑
 * 包括多个步骤和注意事项
 */
function complexLogic() {
  // ...
}

// ❌ 错误：多行注释
// 这是一个多行注释
// 用来解释复杂的业务逻辑
// 包括多个步骤和注意事项
// 但忘记添加空行分隔
function complexLogic() {
  // ...
}
```

### 4.4 JSDoc注释（强制）

**规则**: 所有公共函数、类、方法必须使用JSDoc注释。

**理由**:

- 提供类型检查
- 生成API文档
- 提高代码智能提示

**JSDoc基本格式**:

```javascript
/**
 * 函数描述
 * @param {类型} 参数名 - 参数描述
 * @returns {类型} 返回值描述
 */
```

**示例**:

```javascript
// ✅ 正确：JSDoc注释
/**
 * 计算两个数的和
 * @param {number} a - 第一个数
 * @param {number} b - 第二个数
 * @returns {number} 两个数的和
 */
function add(a, b) {
  return a + b;
}
```

**常用JSDoc标签**:
| 标签 | 说明 | 示例 |
|------|------|------|
| `@param` | 函数参数 | `@param {string} name - 用户名` |
| `@returns` | 返回值 | `@returns {boolean} 是否成功` |
| `@throws` | 可能抛出的异常 | `@throws {Error} 当...时抛出异常` |
| `@example` | 使用示例 | `@example getUserInfo(123)` |
| `@see` | 参考链接 | `@see {@link https://...}` |
| `@since` | 从哪个版本开始 | `@since v1.0.0` |
| `@deprecated` | 已废弃 | `@deprecated 使用newFunction()替代` |

### 4.5 特殊注释标记

**规则**: 使用特殊注释标记标识待办事项、临时修复等。

**常用标记**:
| 标记 | 含义 | 示例 |
|------|------|------|
| `TODO` | 待办事项 | `// TODO: 需要添加错误处理` |
| `FIXME` | 需要修复的Bug | `// FIXME: 这里可能有内存泄漏` |
| `HACK` | 临时修复/折中方案 | `// HACK: 临时解决方案，需要重构` |
| `XXX` | 危险代码 | `// XXX: 这段代码性能有问题` |
| `NOTE` | 重要说明 | `// NOTE: 这个方法必须在主线程调用` |

### 4.6 注释位置规范

**规则**: 注释应该放在被解释代码的**上方**，而不是同一行。

**示例**:

```javascript
// ✅ 正确：注释放在代码上方
// 检查用户权限
if (user.hasPermission) {
  // 执行操作
  performAction();
}

// ❌ 错误：注释放在同一行
if (user.hasPermission) {
  // 检查用户权限
  performAction(); // 执行操作
}
```

### 4.7 注释语言规范

**规则**: 注释使用**中文**，专业术语可保留英文。

**示例**:

```javascript
// ✅ 正确：使用中文注释
// 计算用户年龄
const age = currentYear - birthYear;

// 使用Async/Await处理异步
async function fetchData() {
  // ...
}

// ❌ 错误：使用英文注释（项目要求中文）
// Calculate user age
const age = currentYear - birthYear;
```
