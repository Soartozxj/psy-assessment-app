## 🚫 反模式与禁止事项

### 6.1 禁止使用 `var`

**理由**:

- `var` 有变量提升问题
- `var` 是函数作用域，不是块级作用域
- ES6 的 `const` 和 `let` 更安全

**示例**:

```javascript
// ❌ 禁止：使用var
var count = 0; // 错误
var name = 'John'; // 错误

// ✅ 正确：使用const或let
const count = 0;
let name = 'John';
```

### 6.2 禁止使用 `==` 和 `!=`

**规则**: 必须使用严格相等 `===` 和 `!==`，禁止使用 `==` 和 `!=`。

**理由**:

- `==` 会进行隐式类型转换，导致难以发现的Bug
- `===` 比较值和类型，更安全

**示例**:

```javascript
// ❌ 禁止：使用==和!=
if (value == null) { ... }  // 错误：会匹配null和undefined
if (value != 0) { ... }     // 错误：会进行类型转换

// ✅ 正确：使用===和!==
if (value === null) { ... }
if (value !== 0) { ... }
```

### 6.3 禁止使用 `eval()`

**理由**:

- `eval()` 有严重的安全问题（代码注入）
- 性能差（无法优化）
- 难以调试

**示例**:

```javascript
// ❌ 禁止：使用eval()
const code = 'console.log("Hello")';
eval(code); // 错误

// ✅ 正确：使用其他方式
const code = 'console.log("Hello")';
Function(code)(); // 相对安全，但仍不推荐
```

### 6.4 禁止使用 `with` 语句

**理由**:

- `with` 语句使代码难以理解和优化
- 严格模式下禁止使用

**示例**:

```javascript
// ❌ 禁止：使用with语句
with (obj) {
  name = 'John'; // 不清楚是修改obj.name还是全局变量name
}

// ✅ 正确：直接访问属性
obj.name = 'John';
```

### 6.5 禁止修改函数参数

**规则**: 禁止直接修改函数参数，应先复制。

**理由**:

- 修改函数参数会影响调用者（如果传递的是对象）
- 使代码难以理解

**示例**:

```javascript
// ❌ 禁止：直接修改函数参数
function updateUser(user) {
  user.name = 'New Name'; // 错误：会修改原对象
}

// ✅ 正确：先复制再修改
function updateUser(user) {
  const updatedUser = { ...user, name: 'New Name' };
  return updatedUser;
}
```

### 6.6 禁止在全局作用域声明变量

**规则**: 禁止使用全局变量污染命名空间。

**理由**:

- 全局变量容易被意外修改
- 造成命名冲突

**示例**:

```javascript
// ❌ 禁止：在全局作用域声明变量
var globalConfig = {}; // 错误
let currentUser = null; // 错误

// ✅ 正确：使用模块或命名空间
const App = {
  globalConfig: {},
  currentUser: null
};

// 或使用ES6模块
export const globalConfig = {};
export let currentUser = null;
```

### 6.7 禁止同步API调用

**规则**: 在主线程中禁止使用同步API（如 `localStorage.getItem()` 可以，但 `fetch()` 必须用异步）。

**理由**:

- 同步API会阻塞主线程，导致UI卡顿
- 现代浏览器已不推荐使用同步API

**示例**:

```javascript
// ❌ 禁止：使用同步fetch（假设存在）
const response = fetchSync('/api/data'); // 错误：阻塞主线程

// ✅ 正确：使用异步fetch
const response = await fetch('/api/data');
```

### 6.8 禁止内存泄漏

**规则**: 必须清理定时器、事件监听器、订阅等。

**理由**:

- 内存泄漏会导致应用变慢、崩溃

**示例**:

```javascript
// ❌ 禁止：不清理定时器
function startTimer() {
  setInterval(() => {
    console.log('Timer tick');
  }, 1000); // 错误：定时器永远不会被清理
}

// ✅ 正确：清理定时器
function startTimer() {
  const timerId = setInterval(() => {
    console.log('Timer tick');
  }, 1000);

  // 在适当的时候清理
  return () => clearInterval(timerId);
}

// ❌ 禁止：不清理事件监听器
function addEventListener() {
  window.addEventListener('resize', () => {
    console.log('Window resized');
  }); // 错误：事件监听器永远不会被移除
}

// ✅ 正确：清理事件监听器
function addEventListener() {
  const handler = () => {
    console.log('Window resized');
  };

  window.addEventListener('resize', handler);

  // 在适当的时候清理
  return () => window.removeEventListener('resize', handler);
}
```

**参考**: 内存泄漏详细防治策略参见《Skills架构改造-异常处理与日志规范》第4.3节"安全规范"

## ✅ 代码审查清单

### 7.1 代码风格检查清单

审查代码时，请检查以下项目：

#### 7.1.1 格式化规则

- [ ] 缩进是否使用2个空格
- [ ] 是否统一使用单引号
- [ ] 是否每条语句都以分号结尾
- [ ] 操作符两侧是否有空格
- [ ] 逗号后是否有空格
- [ ] 行长是否不超过100个字符
- [ ] 是否使用K&R风格的大括号

#### 7.1.2 命名规范

- [ ] 变量是否使用camelCase
- [ ] 常量是否使用UPPER_SNAKE_CASE
- [ ] 函数是否使用camelCase + 动词前缀
- [ ] 类是否使用PascalCase
- [ ] 文件名是否符合规范（kebab-case）
- [ ] 事件名是否使用kebab-case

#### 7.1.3 注释规范

- [ ] 公共函数是否有JSDoc注释
- [ ] 复杂逻辑是否有注释解释
- [ ] 注释是否解释"为什么"而不是"是什么"
- [ ] 特殊注释标记（TODO、FIXME等）是否合理使用

#### 7.1.4 最佳实践

- [ ] 是否使用 `const`/`let` 而不是 `var`
- [ ] 是否使用模板字符串而不是 `+` 拼接
- [ ] 是否使用解构赋值
- [ ] 是否使用箭头函数
- [ ] 是否使用 `async/await` 而不是 `.then()`
- [ ] 是否有适当的错误处理（`try-catch`）

#### 7.1.5 反模式检查

- [ ] 是否使用了 `var`（禁止使用）
- [ ] 是否使用了 `==` 或 `!=`（禁止使用）
- [ ] 是否使用了 `eval()`（禁止使用）
- [ ] 是否修改了函数参数
- [ ] 是否有内存泄漏风险（未清理定时器、事件监听器等）

### 7.2 自动化检查工具

**推荐工具**:

1. **ESLint** - JavaScript代码检查工具
   - 配置文件: `.eslintrc.js`
   - 规则: 基于本规范定制

2. **Prettier** - 代码格式化工具
   - 配置文件: `.prettierrc`
   - 自动格式化代码

3. **EditorConfig** - 编辑器配置统一
   - 配置文件: `.editorconfig`
   - 确保不同编辑器的缩进、换行符等一致

**示例配置**:

`.eslintrc.js`:

```javascript
module.exports = {
  parserOptions: {
    ecmaVersion: 2020,
    sourceType: 'module'
  },
  rules: {
    // 强制使用单引号
    quotes: ['error', 'single'],

    // 强制分号
    semi: ['error', 'always'],

    // 强制使用const/let
    'no-var': 'error',

    // 强制使用===和!==
    eqeqeq: 'error',

    // 禁止eval()
    'no-eval': 'error',

    // 最大行长度
    'max-len': ['error', { code: 100 }]

    // ... 其他规则
  }
};
```

`.prettierrc`:

```json
{
  "singleQuote": true,
  "semi": true,
  "tabWidth": 2,
  "printWidth": 100,
  "trailingComma": "es5"
}
```

`.editorconfig`:

```ini
[*.js]
indent_style = space
indent_size = 2
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true
```
