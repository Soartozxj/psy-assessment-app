## 产品概述

优化 Skills 架构改造的规则体系，解决12个规则文件导致的AI注意力分散问题。通过将通用代码风格规范移出 `.codebuddy/rules/` 目录，将规则文件从12个减少到7个（降低42.8%），提升AI对架构红线的执行精度。

## 核心功能

- 将5个通用代码风格规范文件移出 `.codebuddy/rules/` 目录到备份位置
- 更新架构改造规范的引用，指向备份目录中的通用规范
- 更新规则索引文件，说明调整原因和优化效果
- 确保架构改造的7个核心规则文件保持可访问性
- 创建备份目录的README说明文件

## Tech Stack Selection

**当前技术栈**：

- 规则文件格式：Markdown (.md)
- 规则加载机制：CodeBuddy AI Agent 自动加载 `.codebuddy/rules/` 目录下的所有规则文件
- 字符限制：单个规则文件 ≤ 10,000 字符

**技术方案**：

- 文件移动操作（mv命令）
- Markdown文件编辑（更新引用）
- 目录结构优化

## Implementation Approach

### 核心策略：规则体系精简

**问题分析**：

1. **注意力稀释效应**：12个规则文件同时加载，AI模型会出现"丢失在中间"效应（Lost in the Middle），导致架构红线规则被忽略
2. **规则冲突**：命名规范、事件命名在两组文件中都有定义，优先级不明确
3. **上下文溢出**：92,000字符占用大量上下文窗口，导致执行精度下降

**解决方案**：

- **保留**：架构改造规范（7个文件）在 `.codebuddy/rules/` 目录（核心规则）
- **移出**：通用代码风格规范（5个文件）到 `.codebuddy/备份-通用代码风格规范/` 目录（参考规则）
- **优化效果**：规则文件数量从12个减少到7个（降低42.8%），字符总数从92,000减少到54,000（降低41.3%）

### 注意力机制优化原理

**AI注意力分配机制**：

1. **优先关注前面的内容**（锚定效应）
2. **忽略中间内容**（丢失在中间效应）
3. **重点关注最近的指令**（递归效应）

**当前问题**：

```
调整前（12个文件，92,000字符）：
[架构核心规范] → [代码结构规范] → [迁移标准规范] → [异常处理规范]
[改造禁止事项规范] → [检查清单规范] → [快速参考规范]
[核心原则与格式化] → [命名约定] → [注释与文档标准] → [语言特定最佳实践] → [反模式与禁止事项]

问题：架构红线（前7个文件）可能被忽略，通用规范（后5个文件）占用过多注意力
```

**调整后**：

```
调整后（7个文件，54,000字符）：
[架构核心规范] → [代码结构规范] → [迁移标准规范] → [异常处理规范]
[改造禁止事项规范] → [检查清单规范] → [快速参考规范]

优势：AI注意力完全聚焦在架构改造核心规则，架构红线执行精度显著提升
```

## Implementation Notes

### 性能优化

- **上下文窗口占用降低41.3%**：从92,000字符减少到54,000字符
- **规则文件数量降低42.8%**：从12个文件减少到7个文件
- **架构红线可见性提升**：所有架构红线规则都在前7个文件中，不会被忽略

### 向后兼容性

- 通用代码风格规范仍然保留在备份目录中，可作为参考
- 架构改造规范中删除对通用规范的引用，避免冲突
- 开发者仍然可以查阅通用规范，但优先级低于架构改造规范

### 日志记录

- 在规则索引文件中记录调整原因和优化效果
- 在备份目录创建README文件，说明调整方案
