# Web-to-Feishu-Docx Skill

## 概述

本技能将多页面网站内容自动化提取、智能翻译并按层级写入飞书文档，实现「网页 → 本地化文档」的端到端流水线。

**v2.0 新增特性**：

- **智能语言检测**：自动识别源语言，若已是简体中文则询问直接提取还是翻译为其他语言
- **多语言翻译支持**：目标语言不限于简体中文，用户可选择任意语言作为翻译目标
- **飞书文档自动创建**：未提供文档链接时，可自动创建飞书云文档并写入内容

## 工作流程

### 阶段一：发现与索引（Discovery）

- **Step 1.1** — 优先请求 `{baseURL}/llms.txt`，该文件通常包含完整的文档页面索引。
- **Step 1.2** — 若 `llms.txt` 不可用，则抓取首页 HTML，解析左侧导航栏获取所有子页面链接及其标题。
- **Step 1.3** — 生成待处理页面队列，按导航栏原始顺序排列。

### 阶段二：语言检测与决策（Language Detection & Decision）

**⚠️ 此阶段为强制交互步骤，必须在抓取全部内容前完成！**

- **Step 2.0** — 使用 `web_fetch` 抓取首页 HTML，提取 `<html lang="...">` 属性或前 500 字符进行语言检测。

- **Step 2.1** — 根据检测结果分流处理：

  **情况 A：源语言为简体中文（`zh-CN` / `zh`）**
  使用 `ask_followup_question` 弹出交互选择：

  ```
  问题：「检测到目标网站内容为简体中文，请选择处理方式：」
  选项：
    a) 「直接提取原文（不翻译）」
    b) 「翻译为英文（English）」
    c) 「翻译为日文（日本語）」
    d) 「翻译为韩文（한국어）」
    e) 「翻译为繁体中文（zh-TW）」
    f) 「翻译为其他语言（自行输入语言代码）」
  ```

  **情况 B：源语言为其他语言（英文/日文/韩文等）**
  使用 `ask_followup_question` 弹出交互选择：

  ```
  问题：「检测到源语言为 {detected_lang}，请选择目标翻译语言：」
  选项：
    a) 「简体中文（zh-CN）」← 默认选中
    b) 「繁体中文（zh-TW）」
    c) 「英文（English）」
    d) 「日文（日本語）」
    e) 「韩文（한국어）」
    f) 「其他语言（自行输入语言代码）」
    g) 「仅提取原文，不翻译」
  ```

- **Step 2.2** — 等待用户选择后，记录 `target_lang` 和 `skip_translation` 标志，进入内容提取阶段。

### 阶段三：提取与翻译（Extract & Translate）

- **Step 3.1** — 对队列中每个 URL 发起 HTTP GET 请求，获取完整 HTML。
- **Step 3.2** — 内容提取规则：
  - **保留元素**：`<main>`、`<article>`、`.content`、`.markdown-body` 等主内容容器；标题（h1-h6）、段落、列表、代码块、表格、图片 alt 文本、行内代码
  - **过滤元素**：`<nav>`、`<footer>`、`<aside>`、`.sidebar`、`.advertisement`
- **Step 3.3** — 翻译执行规范：
  - 若 `skip_translation=true`：直接输出原文，不做翻译
  - 若 `target_lang` 已指定：翻译为对应语言，语言流畅自然
  - 专业术语按行业惯用译法处理，无统一译法时格式为 `译文（原文）`
  - 代码块内容不翻译，仅保留原始代码
  - URL、文件路径、命令参数不翻译

### 阶段四：结构化写入（Write to Feishu）

- **Step 4.1** — 将翻译后的内容映射为飞书文档块类型：

| 原文元素   | 飞书块类型 | block_type |
| ---------- | ---------- | :--------: |
| H1 标题    | heading1   |     3      |
| H2 标题    | heading2   |     4      |
| H3 标题    | heading3   |     5      |
| 段落       | text       |     2      |
| 无序列表项 | bullet     |     12     |
| 代码块     | code       |     14     |

- **Step 4.2** — 通过飞书开放平台 API 写入文档：

  ```
  POST https://open.feishu.cn/open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children
  ```

  - `document_id`：飞书文档唯一标识（从文档 URL 提取或自动创建后获取）
  - `block_id`：父块 ID（追加模式使用文档根块 ID）
  - `children`：要创建的块数组，每个块包含 `block_type` 和对应内容元素

- **Step 4.3** — 写入完成后获取文档全量块列表，验证块类型分布是否正确。

## 输入参数规范

| 参数                  | 类型    | 必填 | 说明                                                               | 示例                                   |
| --------------------- | ------- | :--: | ------------------------------------------------------------------ | -------------------------------------- |
| `target_url`          | string  |  ✅  | 目标网站首页 URL                                                   | `https://agentskills.io/home`          |
| `feishu_doc_url`      | string  |  ❌  | 目标飞书文档完整 URL。**未提供时将弹出选择：手动填入 或 自动创建** | `https://xxx.feishu.cn/docx/ABC123`    |
| `feishu_token`        | string  |  ❌  | 飞书开放平台 access token（自动创建文档时必填）                    | `t-g1045...`                           |
| `feishu_folder_token` | string  |  ❌  | 飞书云空间文件夹 token，自动创建文档时指定存放位置                 | `fldcnXXXXXX`                          |
| `target_lang`         | string  |  ❌  | 目标语言，**执行前弹出交互选择**，默认 `zh-CN`                     | `zh-CN` / `en` / `ja` / `ko` / `zh-TW` |
| `max_pages`           | number  |  ❌  | 最大抓取页面数，默认无限制                                         | `50`                                   |
| `append_mode`         | boolean |  ❌  | 是否追加模式（true=追加，false=覆盖）                              | `true`                                 |

本技能在三个关键节点需要与用户交互确认，确保输出结果符合预期。

### 决策节点 1：是否需要翻译？（阶段二）

```
  ▼

  │              │
  │ YES          │ NO
  ▼              ▼

• 直接提取原文  • 简体中文（默认）
• 翻译为英文    • 繁体中文
• 翻译为日文    • 英文
• 翻译为韩文    • 日文
• 翻译为繁体    • 韩文
• 其他语言      • 仅提取原文
               • 其他语言
  │              │
  ▼              ▼
记录 target_lang / skip_translation → 进入阶段三
```

### 决策节点 2：写入哪个飞书文档？（阶段四之前）

```
检查 feishu_doc_url 是否已提供？
  │
  │ 已提供 ──► 直接使用
  │
  │ 未提供
  ▼

  问题：「未提供飞书文档链接，请选择处理方式：」
  选项：
    a) 「手动输入飞书文档链接」
    b) 「自动创建新的飞书云文档」
  │
  │ 选 a) ──► 再次弹窗让用户输入 URL
  │
  │ 选 b) ──► 检查 feishu_token 是否可用
  │            │
  │            │ 可用 ──► 调用飞书 API 创建文档
  │            │         POST /open-apis/docx/v1/documents
  │            │         获取 document_id → 继续写入
  │            │
  │            │ 不可用 ──► 提示用户提供 feishu_token
  ▼
继续阶段四：结构化写入
```

### 飞书文档自动创建 API

```http
POST https://open.feishu.cn/open-apis/docx/v1/documents
Authorization: Bearer {feishu_token}
Content-Type: application/json

{
  "title": "{source_title} - 中文翻译"
}
```

返回：

```json
{
  "code": 0,
  "data": {
    "document": {
      "document_id": "docx_XXXXXX",
      "revision_id": 1,
      "title": "..."
    }
  }
}
```

创建后的文档 URL 格式：`https://{domain}.feishu.cn/docx/{document_id}`

## 边界情况处理

| 场景                              | 处理策略                                                           |
| --------------------------------- | ------------------------------------------------------------------ |
| 子页面 URL 为相对路径             | 自动拼接 base URL                                                  |
| 页面返回 404/500                  | 跳过并记录到 `failed_pages`，继续下一个                            |
| 子页面内容超过 10,000 字          | 分批次写入，每批 50 个块                                           |
| 飞书 API 限流（429）              | 指数退避重试，最多 3 次                                            |
| 页面包含动态加载内容              | 若首次抓取不完整，尝试使用浏览器渲染                               |
| 导航栏链接嵌套层级                | 仅展开一级子链接，避免无限深层爬取                                 |
| 源语言无法自动检测                | 默认按英文处理，并在执行前提示用户确认                             |
| 用户选择「其他语言」              | 弹出输入框让用户手动输入 ISO 639-1 语言代码（如 `fr`、`de`、`es`） |
| feishu_token 为空且需自动创建文档 | 提示用户先获取飞书 tenant_access_token                             |
| 自动创建文档失败                  | 列出错误原因，回退到手动输入文档链接方式                           |
| 飞书文档创建成功但无写入权限      | 提示用户在飞书开放平台开启 `docx:document:edit` 权限               |

## 飞书 API 鉴权说明

创建块写入飞书文档需要以下权限（在飞书开放平台应用配置中开启）：

```
docx:document:readonly     — 读取文档结构
docx:document:edit         — 写入文档内容
drive:drive:readonly       — 读取云空间文档信息
```

Token 获取方式：

```
POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal
```

## 代码块处理规范

写入格式：使用飞书原生 `block_type=14`（code）块，保留语言标识和原始内容：

```json
{
  "block_type": 14,
  "code": {
    "elements": [
      {
        "text_run": {
          "content": "<原始代码内容，保留所有换行和缩进>",
          "text_element_style": { "language": <语言代码> }
        }
      }
    ],
    "style": { "language": <语言代码> }
  }
}
```

支持的语言代码（部分）：`PlainText`、`Python`、`JavaScript`、`Bash`、`JSON`、`YAML`、`Markdown`、`SQL`
