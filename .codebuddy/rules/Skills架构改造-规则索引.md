### 规则依赖关系

#### 架构改造规范内部依赖

**优化说明（2026-05-29）**：

- 原"异常处理与日志规范"拆分为3个文件，后合并为1个文件
- 最终优化为8个文件，依赖关系更清晰

```
架构核心规范 (1)
    ↓
代码结构规范 (2)
    ↓
迁移标准规范 (3) ← 依赖 (1)(2)
    ↓
异常处理与日志规范 (4) ← 依赖 (2)
安全规范 (5) ← 依赖 (2)

改造禁止事项规范 (6) ← 依赖 (1)(2)(3)(4)(5)
    ↓
检查清单规范 (7) ← 依赖 (1)-(6)
    ↓
快速参考规范 (8) ← 依赖 (1)-(7)
```

#### 与通用代码风格规范的关系

```
通用代码风格规范 (代码风格规范-coding-style.md)
    ↑ 引用（通用规则）
    |
    +---> 代码结构规范 (2) ← 优先级更高
    |        |
    |        +---> 命名规范（插件类、插件文件等特殊规则）
    |        +---> 代码组织（插件模板）
    |
    +---> 其他架构改造规范 ← 引用通用规则
```

**重要说明**：

1. **优先级**：当架构改造规范与通用代码风格规范冲突时，**以架构改造规范为准**
2. **引用关系**：
   - 架构改造规范引用通用规范中的通用规则（如：变量命名、字符串处理等）
   - 通用规范中添加特殊规则说明，指向架构改造规范中的特定规则
3. **避免重复**：
   - 事件命名规范：以通用规范为准（代码结构规范中已删除冗余内容）
   - 插件类命名规范：以架构改造规范为准（通用规范中已添加引用）

## 🚨 核心原则（不可违反）

### 架构红线（绝对禁止）

❌ **以下行为严格禁止**（详见《架构核心规范》第1.1节）：

1. 禁止绕过PluginBase直接创建插件
2. 禁止使用全局变量污染命名空间
3. 禁止插件间直接依赖
4. 禁止修改核心基类
5. 禁止在H5端使用小程序专有API

### 技术栈约束

✅ **强制技术标准**（详见《架构核心规范》第1.2节）：

- **语言版本**: ES6+ (ES2017+)
- **模块系统**: 动态 `import()`
- **构建工具**: 无 (浏览器原生支持)
- **类型系统**: JSDoc注释 (无TypeScript)
- **浏览器支持**: Chrome 60+, Firefox 55+, Safari 11+
