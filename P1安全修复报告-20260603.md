### FIX-003: 生产环境启用HTTPS

**风险等级**: 🔴 高风险  
**影响文件**:

- `server/psy-api.js`
- `server/.env.example`

**修复方案**:

1. **在 `.env.example` 中添加HTTPS配置项**:

   ```bash
   # ---- HTTPS 配置（生产环境） ----
   # 启用HTTPS（true/false）
   HTTPS_ENABLED=false
   # SSL证书路径（HTTPS_ENABLED=true时使用）
   SSL_CERT_PATH=/path/to/cert.pem
   SSL_KEY_PATH=/path/to/key.pem
   # HTTP自动跳转HTTPS（true/false）
   HTTPS_REDIRECT=false
   ```

2. **HTTPS配置说明**:
   - `HTTPS_ENABLED`: 控制是否启用HTTPS（生产环境应设为 `true`）
   - `SSL_CERT_PATH`: SSL证书文件路径（`.pem` 或 `.crt` 格式）
   - `SSL_KEY_PATH`: SSL私钥文件路径（`.key` 或 `.pem` 格式）
   - `HTTPS_REDIRECT`: 是否将HTTP请求自动重定向到HTTPS

3. **生产环境部署步骤**:
   - 获取SSL证书（推荐使用Let's Encrypt免费证书）
   - 配置Nginx反向代理（推荐）或直接启用Node.js HTTPS
   - 设置 `HTTPS_ENABLED=true` 和正确的证书路径
   - 设置 `HTTPS_REDIRECT=true` 强制HTTPS访问

**Nginx配置示例**（推荐）:

```nginx
server {
    listen 80;
    server_name www.soarto.com.cn soarto.com.cn;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name www.soarto.com.cn soarto.com.cn;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location /api {
        proxy_pass http://localhost:3100;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**Node.js HTTPS配置示例**（备选）:

```javascript
const https = require('https');
const fs = require('fs');

if (process.env.HTTPS_ENABLED === 'true') {
  const options = {
    cert: fs.readFileSync(process.env.SSL_CERT_PATH),
    key: fs.readFileSync(process.env.SSL_KEY_PATH)
  };
  https.createServer(options, app).listen(3443);
  console.log('HTTPS Server running on port 3443');
} else {
  app.listen(PORT);
  console.log('HTTP Server running on port ' + PORT);
}
```
