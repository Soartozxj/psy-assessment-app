> 版本：v1.0 | 更新日期：2026-04-27 | 当前系统版本：v12.9

---

## 一、系统架构概览

```
微信小程序（壳）──web-view──→ H5 SPA（前端/后台）
                                    │
                              cloud-api.js
                                    │
                            ┌───────┴───────┐
                        Nginx 反代        云开发（备用）
                            │
                    Express API (:3100)
                            │
                        MySQL (psy_assessment)
```

**测试覆盖范围**：4 个端 × 13 个前端页面 × 10 个后台模块 × 6 个 JS 模块 × 12 个 API

## 十、技能补充需求

当前测试能力主要依赖 `agent-browser`（浏览器自动化）和现有 Playwright 脚本。以下是需要补充的能力：

### 1. API 自动化测试 ⭐（最急需）
- **当前缺失**：无法自动验证后端 API 响应
- **需要技能**：HTTP 请求工具（curl/fetch 足够，无需额外技能）
- **建议**：写一个 `tests/api-test.cjs`，覆盖上面 12 个 API

### 2. 单元测试框架 ⭐（计分引擎）
- **当前缺失**：计分引擎无单元测试，靠手动验证
- **需要技能**：Node.js 测试框架（内置 `node:test` 或安装 Vitest）
- **建议**：`tests/scoring-engine.test.js`，覆盖上面 12 个用例

### 3. 性能监控
- **当前缺失**：无性能基线数据
- **需要技能**：Lighthouse CI 或 Playwright 性能指标
- **建议**：在 e2e 脚本中收集 `performance.getEntriesByType('navigation')` 数据

### 4. 视觉回归测试（可选）
- **当前缺失**：UI 变更无法自动检测
- **需要技能**：Playwright 截图对比（已有基础，test-screenshots 目录）
- **建议**：关键页面截图基线化

### 5. 真机自动化测试（可选）
- **当前缺失**：无法测试真实微信 WebView 环境
- **需要技能**：微信开发者工具 CLI 或 miniprogram-automator
- **建议**：后续接入 miniprogram-automator 做真机回归
