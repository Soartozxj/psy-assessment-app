# 测试5（删除NPC）修复总结 - 2026-06-01

## 问题描述

测试5（删除NPC）持续失败，原因为：

1. 菜单按钮查找失败 - 菜单按钮默认隐藏（opacity: 0），只在hover时显示
2. 删除按钮查找范围错误 - 在整个document中查找，可能找到错误的下拉菜单
3. 测试数据键名不匹配 - 使用了错误的键名（counselORs）和localStorage键名（npcConfig）

## 修复方案

### 修复1：菜单按钮打开方式（第472-520行）

**原方案**：尝试点击隐藏的菜单按钮（`.cell-menu-btn`）

**新方案**：

- **优先**：直接调用全局函数 `admin.toggleCellMenu(btn)` - 最可靠的方式
- **备用1**：如果`toggleCellMenu`不存在，设置`menuBtn.style.opacity = '1'`后点击
- **备用2**：如果找不到菜单按钮，直接给`.cell-dropdown`添加`.open`类

**关键代码**：

```javascript
if (menuBtn && typeof admin.toggleCellMenu === 'function') {
  log('  使用 toggleCellMenu() 函数打开菜单...', 'info');
  admin.toggleCellMenu(menuBtn);
  await sleep(1000);
}
```

### 修复2：删除按钮查找范围（第525-565行）

**原方案**：在整个`admin.document`中查找删除按钮

**新方案**：

- **优先**：在第一个卡片的`.cell-dropdown`内查找
- **备用**：如果在卡片内找不到，再在全局查找

**关键代码**：

```javascript
const dropdown = firstCard.querySelector('.cell-dropdown');
if (dropdown) {
  deleteBtn = dropdown.querySelector('.dropdown-item.danger');
  // ... 其他选择器
}
```

### 修复3：测试数据键名修正（第819-823行）

**原方案**：

```javascript
const config = { counselORs: testNpcs, ... };
admin.localStorage.setItem('npcConfig', JSON.stringify(config));
```

**新方案**：

```javascript
const config = { counselors: testNpcs, ... };
admin.localStorage.setItem('psy_npc_config', JSON.stringify(config));
```

**关键修正**：

- 数据键名：`counselORs` → `counselors`（小写o，与原代码一致）
- localStorage键名：`npcConfig` → `psy_npc_config`（与原代码`NPC_CONFIG_KEY`常量一致）
- 数据字段：`intro` → `bio`（与原代码一致）

## 技术要点

1. **隐藏元素交互**：对于默认隐藏的元素，不要模拟点击，而是直接调用对应的JavaScript函数
2. **查找范围控制**：操作特定DOM元素时，优先在其内部查找，避免全局查找导致的错误匹配
3. **键名精确匹配**：前端存储的键名必须与原代码完全一致（包括大小写）
