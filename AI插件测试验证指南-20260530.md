## ❌ 常见问题与解决方案

### 问题1：插件初始化失败

**错误信息**：

```
❌ 插件系统初始化失败: AiPlugin is not a constructor
```

**原因**：`ai-plugin.js` 中未正确使用 `export default` 导出类。

**解决方案**：

1. 打开 `mini-app-h5/backend/plugins/core/ai-plugin.js`
2. 检查文件最后是否有 `export default AiPlugin;`
3. 如果没有，添加这行代码
4. 保存文件，刷新浏览器

---

### 问题2：插件实例获取失败

**错误信息**：

```
AI插件实例未找到
```

**原因**：插件未成功注册到 `PluginLoader` 中。

**解决方案**：

1. 打开 `mini-app-h5/backend/plugin-loader.js`
2. 检查 `corePlugins` 对象中是否有 `ai` 配置
3. 确保配置类似这样：
   ```javascript
   const corePlugins = {
     auth: './plugins/core/auth-plugin.js',
     ai: './plugins/core/ai-plugin.js'
     // ...
   };
   ```
4. 保存文件，刷新浏览器

---

### 问题3：配置加载失败

**错误信息**：

```
插件执行失败: Adapter is not defined
```

**原因**：`Adapter` 未正确加载或初始化。

**解决方案**：

1. 检查 `admin-legacy.html` 第12行是否正确引入 `dual-adapter.js`
2. 在Console中执行 `console.log(Adapter);`，检查输出
3. 如果输出 `undefined`，说明 `dual-adapter.js` 未正确加载
4. 检查文件路径是否正确，文件是否存在

---

### 问题4：事件通信失败

**错误信息**：

```
EventHub测试失败: EventHub is not defined
```

**原因**：`EventHub` 未正确加载或初始化。

**解决方案**：

1. 检查 `admin-legacy.html` 第13行是否正确引入 `event-hub.js`
2. 在Console中执行 `console.log(EventHub);`，检查输出
3. 如果输出 `undefined`，说明 `event-hub.js` 未正确加载
4. 检查文件路径是否正确，文件是否存在

---

### 问题5：AI配置页面功能不正常

**现象**：切换到AI配置页面后，页面空白或功能无法使用。

**原因**：旧的 `AIConfig.init()` 调用未完全移除，或新的插件调用方式有误。

**解决方案**：

1. 打开 `admin-legacy.html`
2. 搜索 `AIConfig`，确认没有残留的旧代码
3. 找到切换到AI配置页面的逻辑（约第270行）
4. 确认代码类似这样：
   ```javascript
   if (name === 'aiConfig') {
     setTimeout(() => {
       const aiPlugin = PluginLoader.get('ai');
       if (aiPlugin && typeof aiPlugin.onExecute === 'function') {
         aiPlugin.onExecute({ action: 'loadConfig' });
       }
     }, 100);
   }
   ```
5. 保存文件，刷新浏览器
