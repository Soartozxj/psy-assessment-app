一、处理规则

【0】原文忠实（最高优先级，违反即不合格）

⚠️ 题目内容和选项文本必须逐字照搬手册原文，这是一条不可违反的硬性规则。

具体要求：

- 题目内容：必须与手册原文中的文字完全一致，一个字都不能改
- 选项文本：必须与手册原文中的选项文字完全一致
- 允许的操作：删除题号前缀（如手册写"1.用温和友好的语气与我说话"，提取为"用温和友好的语气与我说话"）；删除题号后的空格和多余标点
- 严禁的操作：改写、意译、简化、润色、替换同义词、用自己的话重述、将口语改为书面语、将长句拆为短句、合并相邻题目

错误示例（以下行为全部禁止）：
手册原文：不想让我长大 -> ❌ 不可改为"使我感到不自在"
手册原文：试图控制我做的每一件事 -> ❌ 不可改为"使我难堪或当众受辱"
手册原文：经常对我微笑 -> ❌ 不可改为"让我感到在家是个受欢迎的成员"
手册原文：似乎不明白我想要什么 -> ❌ 不可改为"信任我"
手册原文：让我觉得自己是可有可无的 -> ❌ 不可改为"对我的私事不感兴趣"
手册原文：从不夸奖我 -> ❌ 不可改为"对我很亲切"

如果你发现手册中的某个题目措辞有语病或不够通顺，也必须原文照搬，不要"修正"它。量表题目是经过心理测量学检验的标准化表述，任何修改都会破坏量表信效度。

【1】多量表识别

手册中可能包含多张量表，常见标识："附："、"附1："、"附2："、"附录一"、"附录二"、"量表一"、"量表二"、或章节标题中的量表名称。

处理方式：

- 逐一识别，每张量表生成一组连续数据行
- 所有量表共用一个表头（只出现一次）
- 每张量表第 1 行填写基本信息，其余题目行留空
- 摘要中说明：共识别到 N 张量表：[名称列表]

【2】版本确认

若手册含多个版本（原版/修订版/简版）且我未指明版本，在摘要中提醒：⚠️ 手册含多个版本，本次基于[XX版本，共N题]，请确认。然后按其中一个版本输出。

【3】题目数量

若我在对话中注明了题目数，严格按该数量提取，不合并不扩展。

【4】反向计分与分值方向

分值方向的核心原则：正向表述的题目，选择"最符合/最严重"应得最高分；反向计分题翻转后也应满足这一原则。

第一步：判断题目表述方向

- 正向表述：题目描述的是"好的/ desirable/ 期望的"的行为、品质或感受。选"非常符合"应得最高分，分值递增（如 0,1,2,3）
  常见正向关键词：疼爱、亲切、温和、友好、关心、了解、信任、鼓励、支持、帮助、表扬、重视、喜欢、耐心
- 负向表述：题目描述的是"不好的/ undesirable/ 不期望的"的行为、品质或感受。这是反向计分题，需要翻转分值
  常见负向关键词：冷漠、漠不关心、缺乏、没有、不自在、难堪、控制、干涉、担心过度
- 判断方法：问自己"如果一个人在这道题上选了非常符合，这个人的状况是好的还是不好的？"好的 -> 正向；不好的 -> 负向（反向计分题）

第二步：仅对负向表述（反向计分题）翻转分值，翻转公式：新分值 = 最小值 + 最大值 - 原始分值

示例1（正向题，不翻转）：题目"对我很亲切"，分值 0,1,2,3。选"非常符合"说明父母很亲切，应得 3 分（最高分）-> 递增，不翻转 ✅
示例2（正向题，不翻转）：题目"给我很多表扬"，分值 0,1,2,3。选"非常符合"说明父母常表扬，应得 3 分 -> 不翻转 ✅
示例3（反向题，需翻转）：题目"对我漠不关心"，原始分值 0,1,2,3 -> 翻转后填 3,2,1,0。选"非常符合"说明父母漠不关心，应得 0 分（最低分）-> 需要翻转 ✅
示例4（反向题，需翻转）：题目"使我感到什么事都做不好"，原始分值 0,1,2,3 -> 翻转后填 3,2,1,0
示例5（⚠️ 常见错误）：题目"用温和友好的语气与我说话"是正面表述（"温和友好"=好品质），不应翻转。如果翻转成 3,2,1,0 就错了，应该是 0,1,2,3
示例6（⚠️ 常见错误）：题目"对我很亲切"是正面表述（"亲切"=好品质），不应翻转。翻转成 3,2,1,0 就错了

第三步：所有题目的"反向计分"列统一填"否"（因为翻转已由你预处理完成）

注意：仅翻转手册原文中明确标注或根据表述方向明确判断为反向计分的题目。不要翻转正向表述的题目。

【5】特殊题型处理（v3.0 新增）

除标准单选题外，系统还支持以下两种特殊题型：

■ 矩阵题（matrix）
特征：一道题包含多个评分项目（行），每个项目共用同一组选项（列）。
常见场景：对多个行为/感受/症状分别评分的量表。

识别规则：满足以下任一条件即判定为矩阵题：

- 手册以表格形式呈现题目（行=评分项，列=选项等级）
- 题目描述为"请对以下各项..."、"以下各项目..."等
- 多个评分项目共用同一组选项

CSV 格式：

- 题型列填 "matrix"
- 选项列填写所有行共用的一组选项（和标准题一样）
- 题目内容填写矩阵题的总体描述文字
- 行项目列（JSON）填写所有行项目，格式为 JSON 数组字符串：
  [{"id":"row_1","label":"项目一"},{"id":"row_2","label":"项目二"},{"id":"row_3","label":"项目三"}]
  注意：id 用 row_1, row_2, ... 格式，label 填写项目原文
- 一道矩阵题占一行 CSV，只有一个题号
- 维度列填写该矩阵题所属的维度

■ 父子题（parent-child）
特征：一道题先有一个主选项（通常是有无/是否），选择特定选项后展开子选项列表（多选）。
常见场景：压力源清单、症状筛查、社会支持来源等。

识别规则：满足以下任一条件即判定为父子题：

- 题目分为两层：先选"有/无"或"是/否"，再从列表中多选具体项目
- 手册明确说"如果选择有，请继续选择以下..."、"请勾选所有适用的项目"

CSV 格式：

- 题型列填 "parent-child"
- 选项列填写主选项（通常只有 2 个：无=0分，有=1分）
- 题目内容填写父题的描述文字
- 子选项列（JSON）填写所有子选项，格式为 JSON 数组字符串：
  [{"id":"sub_1","label":"工作压力"},{"id":"sub_2","label":"家庭矛盾"},{"id":"sub_3","label":"健康问题"}]
  注意：id 用 sub_1, sub_2, ... 格式，label 填写子选项原文
- 一道父子题占一行 CSV，只有一个题号
- 维度列填写该父子题所属的维度

■ 标准单选题（不需要特殊处理）
如果量表全部是标准单选题，题型列留空即可。

⚠️ **分组下拉题（grouped）不支持 CSV 导入**：CSV 表格格式无法完整表达 grouped 题型的分组选项、\_sharedGroups、formula:multiply、allowRepeat 等复杂结构。如量表中包含 grouped 题型，请在摘要中明确说明："⚠️ 本量表含分组下拉题（grouped），CSV 格式无法完整表达，建议使用 JSON 格式导入"。

【6】异常检测（必须执行）

逐一检查以下 6 类异常，发现任何一项都必须在摘要中报告：

异常类型 判定条件 处理方式
─────────────────────────────────────────────────────────────────
编号缺失 题号不连续（如 5 后直接 7） 跳过缺失题（不编造），报告中说明位置
编号重复 同一题号出现多次 后续重编为顺序编号，报告
编号错乱 题号顺序不对但内容完整 按内容顺序重编 1,2,3...，列出原->新映射
内容残缺 有题号但文字缺失或不完整 如实输出现有内容，报告
选项缺失 题目存在但选项不完整 如实输出，空缺留空，报告
维度不明 手册有维度列表但部分题无法归属 按内容推断，列出不确定的题号

报告格式示例：
⚠️ [量表名称] 题目异常：

- 缺失：第 7、13 题在原文中未找到，已跳过
- 重复：第 9 题出现两次，第二次重编为第 10 题
- 内容残缺：第 12 题仅有选项无题目文字

二、CSV 格式规范

【表头】

固定顺序如下，但选项列数按实际最多选项数决定（不固定为 5 个）：

量表名称,量表简称,量表编码,所属分类,图标,主题色,预计时长(分钟),适用人群,量表描述,指导语,标签,状态,排序权重,题号,题目内容,维度,题型,反向计分,行项目(JSON),子选项(JSON),选项A文本,选项A分值,选项B文本,选项B分值,[...按需继续到选项Z]

规则：

- 若最多只有 2 个选项，表头到"选项B分值"即止
- 若最多有 7 个选项，表头延伸到"选项G分值"
- 以所有量表所有题目中最多的选项数确定总列数
- 题型、行项目(JSON)、子选项(JSON) 是 v3.0 新增列

【字段说明】

列名 必填 规则
─────────────────────────────────────────────────────────────────
量表名称 是 完整名称，如"PHQ-9 抑郁症筛查量表"
量表简称 否 缩写，如"PHQ-9"
量表编码 是 仅字母+数字+下划线，全大写，如"PHQ9"、"FACES_CV"
所属分类 是 只能填以下值之一：anxiety / depression / personality / comprehensive / stress / unknown。不要自行编造其他值（如 family、children、sleep 等）
图标 否 单个 emoji，默认 📋
主题色 否 十六进制，如"#4A90D9"，默认蓝色
预计时长(分钟) 否 数字
适用人群 否 文本
量表描述 否 如含英文逗号须用英文双引号括起
指导语 否 如含英文逗号须用英文双引号括起
标签 否 中文逗号分隔
状态 否 1（上架）或 2（草稿）
排序权重 否 数字，越大越靠前
题号 是 从 1 开始的整数（矩阵题/父子题整体算一道题，占一个题号）
题目内容 是 如含英文逗号须用英文双引号括起
维度 否 维度/因子名称，留空归为未分类
题型 否 留空（标准单选）/ "matrix"（矩阵题）/ "parent-child"（父子题）
反向计分 固定 填"否"（分值已预处理翻转）
行项目(JSON) 条件 仅矩阵题填写，格式：[{"id":"row_1","label":"项目名"},...]。非矩阵题留空
子选项(JSON) 条件 仅父子题填写，格式：[{"id":"sub_1","label":"子选项名"},...]。非父子题留空
选项X文本 是 第 X 个选项文字
选项X分值 是 第 X 个选项分数，反向题已翻转

【行规则】

- 每行 = 一道题（矩阵题/父子题整体算一道题，占一行）
- 量表基本信息（前 13 列）只在每张量表的第 1 题行填写，其余行留空（逗号占位）
- 多张量表连续输出，不重复表头

【引号转义】

- ⚠️ 只在字段内容中包含英文逗号（,）、英文双引号（"）或换行符时，才用英文双引号括起该字段。不含这些字符的字段**不要加双引号**。
- 错误示例：用温和友好的语气与我说话 -> """用温和友好的语气与我说话"""（❌ 不含英文逗号，不需要双引号）
- 正确示例：用温和友好的语气与我说话 -> 用温和友好的语气与我说话（✅ 直接写）
- 正确示例：健康, 幸福 -> "健康, 幸福"（✅ 含英文逗号，需要双引号）
- 正确示例：他说"好" -> "他说""好"""（✅ 含英文双引号，需要转义）
- 中文逗号（，）和中文引号（""、''）不需要处理，不需要加英文双引号
- ⚠️ 行项目(JSON) 和 子选项(JSON) 列的内容是 JSON 字符串，如果其中包含英文逗号，同样需要用英文双引号括起
