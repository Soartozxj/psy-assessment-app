## 1. XSS (跨站脚本攻击) 测试

### 1.1 innerHTML 使用检测

**检测结果**: ⚠️ **存在风险**

**风险位置**:

```javascript
// admin-legacy.html 第~820行
msgEl.innerHTML = '<span style="color:var(--danger)">请填写所有字段</span>';

// admin-legacy.html 第~825行
msgEl.innerHTML = '<span style="color:var(--danger)">两次输入的新密码不一致</span>';

// admin-legacy.html 第~830行
msgEl.innerHTML = '<span style="color:var(--success)">✅ ' + result.msg + '</span>';

// admin-legacy.html 第~835行
msgEl.innerHTML = '<span style="color:var(--danger)">❌ ' + result.msg + '</span>';
```

**风险评估**:

- 🟡 **中风险**: 部分 `innerHTML` 使用了 `result.msg` 等动态内容
- 如果 `result.msg` 包含用户输入且未转义，可能导致 XSS 攻击

**修复建议**:

```javascript
// 推荐使用 textContent 替代 innerHTML
msgEl.textContent = '✅ ' + result.msg;

// 如果必须使用 HTML，使用 DOMPurify 库进行净化
import DOMPurify from 'dompurify';
msgEl.innerHTML = DOMPurify.sanitize('<span>...' + result.msg + '</span>');
```

**修复优先级**: **P2** (中优先级)

## 2. 代码注入测试

### 2.1 eval() 使用检测

**检测结果**: 🔴 **高风险**

**风险位置**:

```javascript
// scoring-engine.js 第~145行
function evaluateDerived(derivedMetrics, currentValues) {
  // 按依赖顺序排列，替换指标名后 eval
  // ... eval(code) ...
}

// test-plugins.js 第~12行
eval(pluginBaseCode.replace(/window\./g, 'global.'));

// test-plugins.js 第~25行
eval(npcPluginCode.replace(/typeof\s+document\s*!==\s*['"]undefined['"]/g, 'false').replace(/window\./g, 'global.'));
```

**风险评估**:

- 🔴 **高风险**: `eval()` 可以执行任意代码，如果输入可控，会导致代码注入
- `scoring-engine.js` 中的 `eval()` 用于执行动态生成的计分规则
- `test-plugins.js` 中的 `eval()` 用于测试环境加载插件代码

**修复建议**:

**方案1: 使用 Function 构造函数 + 严格限制**

```javascript
// 替代 eval()
const fn = new Function('metricValues', 'return ' + expression);
return fn(metricValues);
```

**方案2: 使用安全的表达式解析器**

```javascript
// 使用 jsonpath 或 jexl 等安全库
import { Jexl } from 'jexl';
const result = await Jexl.eval(expression, context);
```

**方案3: 白名单验证**

```javascript
// 在执行前验证表达式只包含允许的字符
function isSafeExpression(expr) {
  return /^[a-zA-Z0-9_\.\+\-\*\/\(\)\s]+$/.test(expr);
}
```

**修复优先级**: **P1** (高优先级)

**生产环境建议**:

});
