## 🏗️ 插件架构概述

### 核心概念

**插件系统三大支柱：**

1. **PluginBase** - 所有插件必须继承的基类
   - 提供生命周期管理（init/execute/destroy）
   - 自动管理事件监听器
   - 提供统一的API（showToast/showConfirm等）

2. **PluginLoader** - 插件加载器
   - 核心插件预加载（auth）
   - 可选插件按需加载（scoring/npc/meditation等）
   - 防止重复加载
   - 支持H5和小程序双端

3. **EventHub** - 事件中心
   - 插件间通信
   - 支持事件优先级
   - 支持一次性监听器

### 插件类型

| 类型         | 加载策略               | 示例                     | 说明                       |
| ------------ | ---------------------- | ------------------------ | -------------------------- |
| **核心插件** | 预加载（页面初始化时） | auth                     | 应用启动时必须加载         |
| **可选插件** | 按需加载（用户触发时） | scoring, npc, meditation | 需要时才加载，减少首屏时间 |

## 🔰 创建第一个插件

### 步骤1 - 创建插件文件

在 `plugins/optional/` 目录下创建 `my-plugin.js`：

```javascript
/**
 * my-plugin.js - 我的自定义插件
 *
 * 大白话解释：
 * - 继承 PluginBase 获得标准接口
 * - 使用 Adapter 适配H5和小程序双端
 *
 * @version 1.0.0
 * @date 2026-06-01
 */

class MyPlugin extends PluginBase {
  /**
   * 构造函数 - 必须调用 super()
   */
  constructor() {
    super({
      name: '我的插件',
      version: '1.0.0',
      description: '插件功能描述'
    });

    // 插件私有属性
    this._data = [];
    this._currentItem = null;
  }

  /**
   * 初始化逻辑（必须实现）
   * 在插件加载后自动调用
   */
  async onInit() {
    console.log('🚀 我的插件开始初始化...');

    try {
      // 1. 加载数据
      await this._loadData();

      console.log('✅ 我的插件初始化完成');
    } catch (error) {
      console.error('❌ 我的插件初始化失败:', error);
      throw error;
    }
  }

  /**
   * 执行逻辑（必须实现）
   * @param {object} params - 执行参数
   */
  async onExecute(params = {}) {
    console.log('🎯 我的插件执行:', params);

    const action = params.action || 'list';

    switch (action) {
      case 'list':
        return this._data;
      case 'get':
        return await this._getItem(params.id);
      case 'create':
        return await this._createItem(params.data);
      case 'update':
        return await this._updateItem(params.id, params.data);
      case 'delete':
        return await this._deleteItem(params.id);
      default:
        throw new Error(`未知的操作: ${action}`);
    }
  }

  /**
   * 销毁逻辑（必须实现）
   * 在插件卸载时自动调用
   */
  async onDestroy() {
    console.log('🗑️ 我的插件开始销毁...');

    // 清理状态
    this._data = [];
    this._currentItem = null;

    console.log('✅ 我的插件销毁完成');
  }

  // ==================== 私有方法 ====================

  /**
   * 加载数据
   */
  async _loadData() {
    console.log('📡 加载数据...');

    // 从存储中读取数据
    const stored = this.Adapter.storage.get('my_plugin_data');
    if (stored) {
      this._data = JSON.parse(stored);
    } else {
      // 使用模拟数据
      this._data = [
        { id: 1, name: '示例1', createTime: new Date().toISOString() },
        { id: 2, name: '示例2', createTime: new Date().toISOString() }
      ];
      this._saveData();
    }

    console.log(`✅ 加载了 ${this._data.length} 条数据`);
  }

  /**
   * 保存数据到存储
   */
  _saveData() {
    this.Adapter.storage.set('my_plugin_data', JSON.stringify(this._data));
  }

  /**
   * 获取单个数据项
   */
  async _getItem(id) {
    return this._data.find((item) => item.id === id) || null;
  }

  /**
   * 创建数据项
   */
  async _createItem(data) {
    console.log('➕ 创建数据项:', data.name);

    const newItem = {
      id: Date.now(),
      ...data,
      createTime: new Date().toISOString()
    };

    this._data.push(newItem);
    this._saveData();
    this.showToast('创建成功', 'success');
    return newItem;
  }

  /**
   * 更新数据项
   */
  async _updateItem(id, data) {
    console.log('✏️ 更新数据项:', id);

    const index = this._data.findIndex((item) => item.id === id);
    if (index === -1) {
      throw new Error(`数据项不存在: ${id}`);
    }

    this._data[index] = { ...this._data[index], ...data };
    this._saveData();
    this.showToast('更新成功', 'success');
    return this._data[index];
  }

  /**
   * 删除数据项
   */
  async _deleteItem(id) {
    console.log('🗑️ 删除数据项:', id);

    const index = this._data.findIndex((item) => item.id === id);
    if (index === -1) {
      throw new Error(`数据项不存在: ${id}`);
    }

    this._data.splice(index, 1);
    this._saveData();
    this.showToast('删除成功', 'success');
    return { success: true };
  }
}

// 导出到全局（重要！）
window.MyPlugin = MyPlugin;

console.log('✅ my-plugin.js 加载完成');
```

## 🔄 插件生命周期

### 生命周期图示

```
┌─────────────────────────────────────────────────────┐
│                  插件生命周期                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────┐     ┌─────────┐     ┌─────────┐    │
│  │ 创建实例 │────▶│  初始化  │────▶│  执行    │    │
│  │ constructor│   │  onInit  │     │ onExecute│    │
│  └─────────┘     └─────────┘     └─────────┘    │
│                                         │          │
│                                         ▼          │
│                                    ┌─────────┐     │
│                                    │  销毁    │     │
│                                    │ onDestroy│    │
│                                    └─────────┘     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 详细解释

1. **创建实例（constructor）**
   - 调用 `super()` 传入插件元信息
   - 初始化插件私有属性
   - **不要**在这里进行异步操作

2. **初始化（onInit）**
   - 在插件加载后自动调用
   - 适合加载数据、建立连接等初始化操作
   - 可以抛出异常，阻止插件加载

3. **执行（onExecute）**
   - 通过 `PluginLoader.execute()` 调用
   - 根据 `action` 参数执行不同操作
   - 支持异步操作

4. **销毁（onDestroy）**
   - 在插件卸载时自动调用
   - 清理状态、取消事件监听、关闭连接
   - 必须幂等（多次调用不会出错）

## 📡 事件通信

### EventHub API

```javascript
// 订阅事件
const unsubscribe = EventHub.on('event-name', (data) => {
  console.log('收到事件:', data);
});

// 订阅一次性事件
const unsubscribeOnce = EventHub.once('event-name', (data) => {
  console.log('收到一次性事件:', data);
});

// 发射事件
EventHub.emit('event-name', { message: 'hello' });

// 取消订阅
unsubscribe();
unsubscribeOnce();
```

### 插件间通信示例

**插件A发送事件：**

```javascript
// 在插件A中
async onExecute(params) {
  // 执行某些操作...

  // 发送事件通知其他插件
  if (window.EventHub) {
    await window.EventHub.emit('data-updated', {
      plugin: 'plugin-a',
      action: 'create',
      data: newData
    });
  }

  return result;
}
```

**插件B监听事件：**

```javascript
// 在插件B的onInit中
async onInit() {
  // 监听插件A的事件
  if (window.EventHub) {
    window.EventHub.on('data-updated', (data) => {
      console.log('收到数据更新通知:', data);
      // 执行相应操作...
    });
  }
}
```

### 标准事件命名规范

| 事件名            | 触发时机             | 数据格式                         |
| ----------------- | -------------------- | -------------------------------- |
| `plugin-loaded`   | 插件加载完成         | `{ name, version, description }` |
| `plugin-unloaded` | 插件卸载完成         | `{ name }`                       |
| `plugins-ready`   | 所有核心插件加载完成 | `{ corePlugins, loadedCount }`   |
| `data-updated`    | 数据更新             | `{ plugin, action, data }`       |
| `user-login`      | 用户登录             | `{ userId, username }`           |
| `user-logout`     | 用户登出             | `{}`                             |

## 📱 双端适配

### Adapter API

`Adapter` 提供统一的双端接口，屏蔽H5和小程序的差异。

#### Storage（存储）

```javascript
// 设置存储
Adapter.storage.set('key', 'value');

// 获取存储
const value = Adapter.storage.get('key');

// 删除存储
Adapter.storage.remove('key');

// 清空存储
Adapter.storage.clear();
```

#### HTTP（网络请求）

```javascript
// GET 请求
const result = await Adapter.http.get('/api/data');

// POST 请求
const result = await Adapter.http.post('/api/data', { name: 'test' });

// PUT 请求
const result = await Adapter.http.put('/api/data/1', { name: 'updated' });

// DELETE 请求
const result = await Adapter.http.delete('/api/data/1');
```

#### UI（用户界面）

```javascript
// 显示Toast提示
this.showToast('操作成功', 'success');

// 显示确认弹窗（Promise风格）
const confirmed = await this.showConfirm('确定要删除吗？', {
  title: '确认删除',
  dangerMode: true
});

if (confirmed) {
  // 用户点击了确认
}
```

#### Logger（日志）

```javascript
// 普通日志
this.Logger.info('这是一条信息日志');

// 警告日志
this.Logger.warn('这是一条警告日志');

// 错误日志
this.Logger.error('这是一条错误日志');
```

## 💡 最佳实践

### 1. 错误处理

```javascript
async onExecute(params) {
  try {
    // 参数验证
    if (!params.action) {
      throw new Error('action 参数是必需的');
    }

    // 执行操作
    const result = await this._doSomething(params);

    return result;
  } catch (error) {
    // 记录错误日志
    this.Logger.error('执行失败:', error);

    // 显示错误提示
    this.showToast(error.message, 'error');

    // 重新抛出异常
    throw error;
  }
}
```

### 2. 参数验证

```javascript
async _createItem(data) {
  // 验证必需字段
  if (!data.name) {
    throw new Error('name 字段是必需的');
  }

  // 验证数据类型
  if (typeof data.name !== 'string') {
    throw new Error('name 必须是字符串');
  }

  // 验证数据范围
  if (data.name.length > 50) {
    throw new Error('name 长度不能超过50个字符');
  }

  // 执行业务逻辑
  // ...
}
```

### 3. 事件监听器管理

```javascript
async onInit() {
  // 正确的做法：使用插件基类的事件监听器管理
  this._listeners.push(
    // 监听DOM事件
    this._addEventListener(document, 'click', (e) => {
      // 处理点击事件
    })
  );

  // 监听EventHub事件
  if (window.EventHub) {
    const unsubscribe = window.EventHub.on('data-updated', (data) => {
      // 处理数据更新
    });

    // 将取消订阅函数添加到监听器列表
    this._listeners.push({ remove: unsubscribe });
  }
}

async onDestroy() {
  // 不需要手动清理！基类会自动清理 this._listeners
  // 只需要清理其他资源（如定时器、连接等）

  if (this._timer) {
    clearInterval(this._timer);
    this._timer = null;
  }
}
```

### 4. 异步操作

```javascript
async onExecute(params) {
  // 正确的做法：使用 async/await
  try {
    this.showToast('开始处理...', 'info');

    // 异步操作1
    const result1 = await this._step1(params);

    // 异步操作2
    const result2 = await this._step2(result1);

    // 异步操作3
    const result3 = await this._step3(result2);

    this.showToast('处理完成', 'success');
    return result3;
  } catch (error) {
    this.showToast('处理失败: ' + error.message, 'error');
    throw error;
  }
}
```

### 5. 数据安全

```javascript
// 敏感数据不要存储在localStorage
// ❌ 错误做法
Adapter.storage.set('user-password', password);

// ✅ 正确做法：使用内存变量或安全存储
this._password = password; // 内存存储

// 或者对于小程序，使用微信的加密存储
if (typeof wx !== 'undefined') {
  wx.setStorage({
```

## ❓ 常见问题

### Q1: 插件无法加载？

**可能原因：**

1. 插件类没有正确导出到全局

   ```javascript
   // ✅ 正确做法
   window.MyPlugin = MyPlugin;

   // ❌ 错误做法
   export default MyPlugin; // 这会导致PluginLoader找不到插件类
   ```

2. 插件路径配置错误

   ```javascript
   // 检查 plugin-loader.js 中的 pluginBaseUrl
   this.pluginBaseUrl = '/mini-app-h5/backend/plugins/';
   ```

3. 插件类名不匹配
   ```javascript
   // PluginLoader会查找 window.MyPlugin
   // 所以类名必须是 MyPlugin
   class MyPlugin extends PluginBase { ... }
   ```

### Q2: 插件加载后无法执行？

**可能原因：**

1. `onInit()` 方法抛出异常
   - 检查控制台错误日志
   - 确保异步操作正确处理

2. 插件没有正确继承 PluginBase

   ```javascript
   // ✅ 正确做法
   class MyPlugin extends PluginBase {
     async onInit() { ... }
     async onExecute(params) { ... }
     async onDestroy() { ... }
   }

   // ❌ 错误做法：缺少必要方法
   class MyPlugin extends PluginBase {
     // 缺少 onInit, onExecute, onDestroy
   }
   ```

### Q3: 如何在H5和小程序双端运行？

**解决方案：使用Adapter**

```javascript
// ✅ 正确做法：使用Adapter
// 存储
Adapter.storage.set('key', 'value');

// 网络请求
const result = await Adapter.http.get('/api/data');

// UI提示
this.showToast('提示', 'info');

// ❌ 错误做法：直接使用平台特定API
// H5特定
localStorage.setItem('key', 'value');

// 小程序特定
wx.setStorage({ key: 'key', data: 'value' });
```

### Q4: 如何调试插件？

**调试技巧：**

1. 在插件中添加详细日志

   ```javascript
   async onInit() {
     console.log('🚀 插件开始初始化...');
     console.log('   Adapter:', this.Adapter);
     console.log('   Logger:', this.Logger);
     // ...
   }
   ```

2. 使用 PluginLoader 的调试功能

   ```javascript
   // 在控制台中调用
   PluginLoader._debug();
   ```

3. 检查插件状态

   ```javascript
   // 查看已加载的插件
   console.log(PluginLoader.plugins);

   // 查看插件信息
   console.log(PluginLoader.getPluginsInfo());
   ```

### Q5: 插件执行失败，如何排查？

**排查步骤：**

1. 检查参数是否正确

   ```javascript
   async onExecute(params) {
     console.log('收到的参数:', params);
     // ...
   }
   ```

2. 检查返回值是否正确

   ```javascript
   const result = await someAsyncOperation();
   console.log('操作结果:', result);
   return result;
   ```

3. 使用 try-catch 捕获异常
   ```javascript
   try {
     const result = await this._doSomething(params);
     return result;
   } catch (error) {
     console.error('执行失败:', error);
     console.error('错误堆栈:', error.stack);
     throw error;
   }
   ```
