# 系统提示词与 Skill 架构优化实施方案

**文档版本**: v1.1  
**制定日期**: 2026-06-03  
**更新日期**: 2026-06-03  
**目标**: 将现有系统提示词完全迁移到 Skill 插件架构，提升系统的可维护性、可扩展性和智能化水平

---

## 一、当前状态分析

### 1.1 系统架构现状

| 模块       | 实现方式                      | 架构状态 | 对接状态                          |
| ---------- | ----------------------------- | -------- | --------------------------------- |
| 提示词管理 | localStorage + IIFE           | 混合架构 | ✅ 部分对接（新增版本同步到 API） |
| 量表管理   | 内联代码（admin-legacy.html） | 单体架构 | 🟡 插件已创建，未完全对接         |
| AI 配置    | 内联代码                      | 单体架构 | 🟡 插件已创建，未完全对接         |
| 认证系统   | 内联代码                      | 单体架构 | ❌ 未改造                         |
| 事件系统   | admin-event-delegation.js     | 改进中   | ✅ 已对接（覆盖率 80%）           |
| 数据看板   | Chart.js                      | 改进中   | ✅ 已对接                         |
| Toast 提示 | showToast 函数                | 改进中   | ✅ 已优化（支持堆叠显示）         |

### 1.2 存在问题

1. **架构不统一**：部分功能使用 Skill 架构，部分仍使用传统方式
2. **数据持久化不足**：提示词数据存储在 localStorage，易丢失
3. **模块化程度低**：核心文件 `admin-legacy.html` 仍有 24768 行（虽已拆分为 15 个模块）
4. **Skill 同步不完整**：新增版本时同步到 API，但编辑/删除未同步
5. **缺少 Skill 管理界面**：无法直观查看、编辑、测试 Skill
6. **文档不完善**：缺少 Skill 开发指南和用户手册

---

## 二、优化目标

### 2.1 总体目标

将系统提示词完全迁移到 Skill 插件架构，实现：

- ✅ **架构统一**：所有功能模块化为 Skill 插件
- ✅ **数据持久化**：使用后端 API + 数据库存储
- ✅ **模块化**：每个插件独立文件，职责单一
- ✅ **可扩展性**：支持热插拔、依赖管理、版本控制
- ✅ **可维护性**：支持单元测试、集成测试
- ✅ **智能化**：支持上下文感知、AI 动态生成提示词

### 2.2 具体目标

| 目标       | 描述                           | 衡量指标                 |
| ---------- | ------------------------------ | ------------------------ |
| 架构统一   | 所有功能模块化为 Skill 插件    | 插件化率 100%            |
| 数据持久化 | 使用后端 API + MySQL 存储      | 数据不丢失，支持多端同步 |
| 模块化     | 每个插件 500-1500 行，职责单一 | 单文件不超过 1500 行     |
| 可扩展性   | 支持热插拔、依赖管理           | 添加新功能无需修改主文件 |
| 可维护性   | 支持单元测试、集成测试         | 测试覆盖率 > 80%         |
| 智能化     | 支持上下文感知、AI 动态生成    | 提示词根据上下文动态调整 |
