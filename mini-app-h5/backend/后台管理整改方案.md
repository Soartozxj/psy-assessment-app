## 二、当前架构

```
侧栏菜单                        实际复杂度 / 问题
═══════════════════════════════════════════════════════════
📊 数据看板                     · 手写 SVG 图表，无时间筛选
📋 量表管理  ← 🔴 最臃肿        · 4 Tab × N 功能，弹窗 700+ 行 HTML
    ├ 量表列表                  · 搜索/筛选/排序/新建/导入/导出/AI配置
    ├ 量表类型                  · 分类管理/图标选择器(80+ emoji)/颜色
    ├ 计分规则                  · 表+弹窗编辑器/导入AI JSON/测试
    └ 测评详情                  · 量表列表+Prompt编辑器+AI生成
❓ 题目库                        · 占位符（跳转提示）
👥 用户列表                     · 空壳骨架
📈 测评记录                     · 空壳骨架
💬 用户评价                     · 3 Tab（数据/标签/配置）
🎭 NPC场景配置                  · 3 Tab + 3个云端同步按钮
⚙️ 系统设置                     · 内容极少（仅2张卡片）
🛡️ 系统运维                     · 混合 AI配置/监控/手册，3个完全不同领域
💡 系统提示词                   · 多版本管理 + 使用流程编辑器
```

**数据流：**

```
window.scales (全局变量)
    ↕ syncToFrontend() 手动同步
    ├── localStorage(psy_scales_data)  ← loadScales() 读取
    ├── localStorage(psy_scales_synced) ← 前端页面读取
    ├── LNMP API (PUT /api/scales-json) ← 服务器持久化
    └── CloudBase (CloudData.syncScales) ← 小程序读取

问题：五处数据源，一处不同步即导致不一致
```
