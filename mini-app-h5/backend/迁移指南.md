## 🎯 迁移目标

把现有的 **15个JS模块** 逐步改造为 **插件化架构**

### 改造前后对比

| 对比项         | 改造前                   | 改造后                           |
| -------------- | ------------------------ | -------------------------------- |
| **代码组织**   | 15个独立JS文件，互相依赖 | 插件化，每个功能一个插件         |
| **加载方式**   | 全部`<script>`标签引入   | 核心插件预加载，可选插件按需加载 |
| **代码复用**   | 低（复制粘贴多）         | 高（插件可复用）                 |
| **双端支持**   | 只支持H5                 | H5和小程序都能用                 |
| **新手友好度** | 低（代码乱，难懂）       | 高（结构清晰，注释详细）         |

#### 步骤2.4：把原有 `AIConfig` 代码迁移到插件

**原代码**（`admin-ai-config.js`）：

```javascript
var AIConfig = {};

AIConfig.init = function() { ... };
AIConfig.selectProvider = function(provider) { ... };
// ... 其他函数
```

**改造后**（`ai-plugin.js`）：

```javascript
class AIPlugin extends PluginBase {
  async onInit() {
    // 原 AIConfig.init() 的逻辑放在这里
  }

  async selectProvider(provider) {
    // 原 AIConfig.selectProvider() 的逻辑放在这里
  }
}
```

**改造原则**：

1. 原 `AIConfig.xxx()` 函数 → 改成 `this.xxx()` 方法
2. 原 `var currentConfig = {}` 变量 → 改成 `this.config` 或 `this.xxx`
3. 原 `showToast()` 调用 → 改成 `this.showToast()`
4. 原 `localStorage.getItem()` 调用 → 改成 `Adapter.storage.get()`

**示例对比**：

❌ **改造前**（`admin-ai-config.js`）：

```javascript
function loadConfig() {
  try {
    var saved = localStorage.getItem('psy_ai_config');
    if (saved) {
      currentConfig = Object.assign({}, DEFAULT_CONFIG, JSON.parse(saved));
    }
  } catch (e) {
    console.warn('Failed to load AI config:', e);
  }
}
```

✅ **改造后**（`ai-plugin.js`）：

```javascript
async onInit() {
  try {
    const saved = Adapter.storage.get('psy_ai_config');
    if (saved) {
      this.config = Object.assign({}, this.getDefaultConfig(), saved);
    }
  } catch (e) {
    Adapter.logger.warn('Failed to load AI config:', e);
  }
}

getDefaultConfig() {
  return {
    provider: 'dashscope',
    dashscope_key: '',
    dashscope_model: 'qwen-turbo',
    deepseek_key: '',
    deepseek_model: 'deepseek-chat'
  };
}
```

## 📝 常见问题解答

### Q1：改造过程中，现有功能还能用吗？

**A**：能！采用**渐进式改造**，改造一个插件，就启用一个插件，没改造的模块继续保持原样。

**示例**：

```html
<!-- admin-legacy.html -->
<script>
  document.addEventListener('DOMContentLoaded', async function() {
    // 只加载已改造完成的插件
    await PluginLoader.load('ai', true);  // AI插件已改造，加载
    // await PluginLoader.load('scale', true);  // 量表插件未改造，暂不加载

    // 未改造的模块继续保持原样
    // <script src="admin-scale-list.js?v=3"></script>  <!-- 保持原样 -->
  });
</script>
```

### Q2：如果改造失败，能回退吗？

**A**：能！因为每个插件是独立的文件，改造失败只需要：

1. 把插件文件的 `export default` 注释掉
2. 恢复原来的 `<script>` 引入方式

**示例**：

```javascript
// ai-plugin.js
// export default AIPlugin;  // 注释掉，禁用插件

// admin-legacy.html
<script src="admin-ai-config.js?v=1"></script>  <!-- 恢复旧代码 -->
```

### Q3：改造后的性能会提升吗？

**A**：会！主要有3个提升：

1. **首屏加载更快**：核心插件预加载，可选插件按需加载
2. **内存占用更少**：不用的插件可以卸载（`PluginLoader.unload('xxx')`）
3. **代码复用更高**：插件可以在H5和小程序两端复用

### Q4：新手能完成改造吗？

**A**：能！只要按照这个指南的**3步走**策略：

1. 第1步：搭建基础设施（复制粘贴即可）
2. 第2步：改造第一个插件（有详细示例）
3. 第3步：逐步改造其他模块（有模板可用）

**预计总时间**：2-3周（每天2-3小时）

## 🎉 总结

这个迁移指南的核心思想是：**渐进式改造，小步快跑**

- ✅ 不要一次性改造所有代码（风险高）
