# XSS 修复指南 - P2安全修复

## 概述

本文档提供系统性的XSS漏洞修复方案，确保所有用户输入和动态内容都经过正确转义。

## 修复原则

### 1. 输出编码规则

| 上下文           | 编码方法                  | 示例                                      |
| ---------------- | ------------------------- | ----------------------------------------- |
| HTML正文         | `escapeHtml()`            | `<div>{userInput}</div>`                  |
| HTML属性         | `escapeHtml()` + 引号包裹 | `<div title="{userInput}">`               |
| JavaScript字符串 | `escapeJs()`              | `<script>var x = "{userInput}";</script>` |
| URL参数          | `encodeURIComponent()`    | `<a href="/path?q={userInput}">`          |
| CSS值            | `escapeCss()`             | `<div style="color: {userInput}">`        |

### 2. 安全的DOM操作

| 不安全方法                                       | 安全替代方案                                    | 说明         |
| ------------------------------------------------ | ----------------------------------------------- | ------------ |
| `element.innerHTML = userInput`                  | `element.textContent = userInput`               | 纯文本场景   |
| `element.innerHTML = '<b>' + userInput + '</b>'` | `element.textContent = userInput` + CSS样式     | 需要样式时   |
| `document.write(userInput)`                      | `document.createElement()` + 设置属性           | 动态创建元素 |
| `eval(userInput)`                                | `JSON.parse(userInput)` 或 `Function()`构造函数 | 解析数据     |

### 3. 使用安全工具库

```javascript
// 引入安全工具库
const SecurityUtils = require('./security-utils.js');

// 1. HTML转义
const safeHtml = SecurityUtils.escapeHtml(userInput);

// 2. 安全的DOM操作
SecurityUtils.safeSetText(element, userInput);

// 3. 验证输入安全性
if (!SecurityUtils.isSafeString(userInput)) {
  // 拒绝危险输入
}

// 4. 清理输入
const cleanInput = SecurityUtils.sanitizeInput(userInput, 500);
```

## 批量修复步骤

### 步骤1: 识别高风险模式

搜索以下模式，按优先级修复：

1. **最高优先级**: `innerHTML =` 包含用户输入
2. **高优先级**: `outerHTML =` 包含用户输入
3. **中优先级**: `document.write()` 或 `document.writeln()`
4. **低优先级**: `eval()` 或 `new Function()`

### 步骤2: 应用修复策略

对于每个识别出的风险点：

1. 如果是显示纯文本 → 使用 `textContent`
2. 如果需要简单格式化 → 使用 `SecurityUtils.safeSetHtml()`
3. 如果必须使用HTML → 对所有变量使用 `escapeHtml()`

### 步骤3: 验证修复效果

1. 手动测试：输入 `<script>alert('XSS')</script>` 验证是否被转义
2. 自动化测试：运行安全测试套件
3. 代码审查：检查是否引入新的问题

## 特定场景修复示例

### 场景1: 显示用户生成的量表名称

**修复前**:

```javascript
document.getElementById('scale-name').innerHTML = scale.name;
```

**修复后**:

```javascript
// 方法1: 使用textContent（推荐）
document.getElementById('scale-name').textContent = scale.name;

// 方法2: 使用SecurityUtils（如果需要格式化）
SecurityUtils.safeSetText(document.getElementById('scale-name'), scale.name);
```

### 场景2: 动态生成表格行

**修复前**:

```javascript
tbody.innerHTML = data.map((item) => `<tr><td>${item.name}</td><td>${item.value}</td></tr>`).join('');
```

**修复后**:

```javascript
// 方法1: 使用textContent（推荐）
tbody.innerHTML = data
  .map(
    (item) => `<tr><td>${SecurityUtils.escapeHtml(item.name)}</td><td>${SecurityUtils.escapeHtml(item.value)}</td></tr>`
  )
  .join('');

// 方法2: 使用DOM API（更安全）
data.forEach((item) => {
  const row = document.createElement('tr');
  const nameCell = document.createElement('td');
  nameCell.textContent = item.name;
  const valueCell = document.createElement('td');
  valueCell.textContent = item.value;
  row.appendChild(nameCell);
  row.appendChild(valueCell);
  tbody.appendChild(row);
});
```

### 场景3: 显示错误消息

**修复前**:

```javascript
document.getElementById('error-msg').innerHTML = `错误: ${error.message}`;
```

**修复后**:

```javascript
const errorMsg = document.getElementById('error-msg');
errorMsg.textContent = `错误: ${error.message}`;
// 或者使用安全工具
SecurityUtils.safeSetText(errorMsg, `错误: ${error.message}`);
```

element.onclick = function () {

## 后端API修复

### 修复API响应

```javascript
// 修复前
app.get('/api/users', (req, res) => {
  res.json({ users: users }); // 直接返回用户输入
});

// 修复后
app.get('/api/users', (req, res) => {
  const safeUsers = users.map((user) => ({
    ...user,
    name: SecurityUtils.escapeHtml(user.name),
    bio: SecurityUtils.escapeHtml(user.bio)
  }));
  res.json({ users: safeUsers });
});
```

## 测试验证

### 手动测试用例

1. **基本XSS测试**: 输入 `<script>alert('XSS')</script>`
2. **属性XSS测试**: 输入 `" onmouseover="alert('XSS')`
3. **URL XSS测试**: 输入 `javascript:alert('XSS')`
4. **CSS XSS测试**: 输入 `expression(alert('XSS'))`

### 自动化测试

```javascript
// 测试SecurityUtils.escapeHtml
console.assert(
  SecurityUtils.escapeHtml('<script>alert("XSS")</script>') ===
    '&lt;script&gt;alert(&quot;XSS&quot;)&lt;&#x2F;script&gt;',
  'escapeHtml应该转义HTML标签'
);

// 测试SecurityUtils.safeSetText
const testElement = document.createElement('div');
SecurityUtils.safeSetText(testElement, '<script>alert("XSS")</script>');
console.assert(testElement.textContent === '<script>alert("XSS")</script>', 'safeSetText应该设置纯文本');
```

```html
<!-- 在HTML头部添加 -->

  http-equiv="Content-Security-Policy"
  content="default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';"
/>
```

1. 定期检查代码中的`innerHTML`使用
2. 使用ESLint插件: `eslint-plugin-xss`
3. 集成到CI/CD: 自动扫描XSS漏洞

## 总结

通过系统性地应用这些修复策略，可以显著降低XSS攻击风险。关键是：

1. **默认使用`textContent`**而不是`innerHTML`
2. **对所有用户输入进行转义**
3. **使用安全工具库**统一处理
4. **定期审查和测试**
